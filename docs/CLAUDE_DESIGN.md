# Claude Design — Phase D Notes

This document captures what the Claude Design research-preview workflow would add to SkyHerd, what is already in-repo, and a runbook for contributors who want to regenerate the design artifacts.

---

## What is in-repo today

The dashboard aesthetic was generated via the Stitch MCP (`/stitch-design`) and hand-refined into production components:

- `docs/design/dashboard.png` — full ops-console dashboard (live production screenshot)
- `docs/design/rancher.png` — /rancher PWA incoming-call screen
- `docs/design/cross-ranch.png` — /cross-ranch mesh handoff view
- `web/src/index.css` — full token system + chip/button/animation primitives
- `web/src/components/` — 7 hand-rolled components (AgentLanes, CostTicker, AttestationPanel, RanchMap, CrossRanchView, RancherPhone, AgentLane)

The design system is fully documented in [DESIGN_SYSTEM.md](DESIGN_SYSTEM.md).

## What the full Claude Design workflow adds

Claude Design (claude.ai/design) is an Anthropic product launched in the same week as this hackathon — powered by Opus 4.7, the same model running the SkyHerd agents. The full research-preview workflow supports:

- **Handoff-bundle export**: structured JSON of every token, component spec, spacing rule, and typography declaration — machine-readable and importable by any design tool
- **Direct-to-Canva export**: one-click publish of screens and component sheets to a Canva document for pitch decks and investor materials
- **PDF / PPTX / HTML export**: presentation-ready renders with correct fonts, spacing, and brand palette baked in
- **Iterative prompt-to-screen generation**: describe a screen change in natural language; Opus 4.7 re-renders the affected components and diffs the token deltas

For a hackathon submission, the headline value is this: every pixel in the dashboard was shaped by an Anthropic product. The agents run on Opus 4.7. The design was generated by Claude (Stitch, which is the pre-release form of Claude Design). The attestation chain verifies it. The full Claude Design path would close the loop by exporting a judge-facing handoff bundle that is itself an Anthropic artifact.

---

## Why this matters for judges

The "Most Creative Opus 4.7 Exploration" prize criterion asks how deeply the submission integrates Anthropic's model capabilities. SkyHerd's answer is: the model runs the ranch, and the model designed the interface that visualizes it. Claude Design's Opus 4.7 power means the design layer is not decorative — it is a second instance of the same model, applying the same reasoning to visual layout that the agents apply to livestock decisions.

## Stretch: post-submission Claude Design regeneration

The intended post-submission step (not blocking the hackathon deadline) is:

1. Open claude.ai/design
2. Load the existing token system from `web/src/index.css` as a context document
3. Prompt: "Generate a complete design handoff bundle for a dark ops-console dashboard with these tokens, these components, and these three reference screenshots"
4. Export the handoff bundle as JSON and commit to `docs/design/handoff-bundle.json`
5. Export a Canva deck and link from the ONE_PAGER
6. Re-export `dashboard.png`, `rancher.png`, `cross-ranch.png` from the Claude Design renderer and replace in `docs/design/` for a purely Anthropic-provenance artifact set

This step is marked as stretch because it requires the Claude Design research preview to be available to the account, which was not confirmed before the deadline.

## Reproducing the design flow (contributor runbook)

```bash
# 1. View current production screenshots
open docs/design/dashboard.png
open docs/design/rancher.png
open docs/design/cross-ranch.png

# 2. Inspect the live token system
cat web/src/index.css          # @theme block = full token registry

# 3. Run the dashboard locally
uv sync && (cd web && pnpm install && pnpm run build)
make dashboard                 # http://localhost:8000

# 4. To regenerate via Stitch (MCP path)
# Invoke /stitch-design in a Claude Code session at repo root.
# Reference docs/design/ screenshots as existing baseline.
# Ask Stitch to apply the token system from index.css.

# 5. To regenerate via Claude Design (research preview)
# Navigate to claude.ai/design
# Upload docs/design/dashboard.png as reference
# Paste web/src/index.css token block as context
# Export handoff bundle → docs/design/handoff-bundle.json
```

The component-level implementation details are in [DESIGN_SYSTEM.md](DESIGN_SYSTEM.md). The visual screenshots are the ground truth; the CSS tokens are the specification.
