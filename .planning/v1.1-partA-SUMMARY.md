# v1.1 Part A — Real Activity Driver (In-Process Ambient Timelapse)

## One-liner

Run all 8 demo scenarios *inside* the server process against the same
mesh/world/ledger the SSE broadcaster reads, so `http://localhost:8000`
shows visible activity out of the box — no second terminal, no MQTT bridge,
no mutation of the byte-identical seed=42 fast path.

## Endpoint contracts

### `POST /api/ambient/speed`

- **Body**: `{ "speed": float }` — range `[0.0, 120.0]`, validated by
  pydantic `Field(..., ge=0.0, le=120.0)`.
- **200**: `{ "speed": <new_speed_float> }`.
- **404**: `{"detail": "ambient driver not attached"}` when
  `app.state.ambient_driver` is unset (i.e. `--no-ambient` or mock mode).
- **Semantics**: takes effect at the next scenario step boundary; no restart.

### `POST /api/ambient/next`

- **No body**.
- **200**: `{ "skipped": "<scenario_name>" }` — echoes the scenario that
  was active at the moment of the call; `null` if the loop was between
  scenarios.
- **404**: same as above.
- **Semantics**: cancels the in-flight `_run_async_shared` task; the driver
  emits `scenario.ended` and the rotation loop advances to the next
  scenario after the normal cooldown.

## SSE surface added (Part B hook)

Every scenario transition in the ambient loop broadcasts two events via the
shared `EventBroadcaster` (duck-typed: `emit` → `publish` → `_broadcast`):

```
event: scenario.active
data:  { "name": "coyote",
         "pass_idx": 0,
         "speed": 15.0,
         "started_at": "2026-04-22T12:34:56.789123+00:00" }

event: scenario.ended
data:  { "name": "coyote",
         "pass_idx": 0,
         "outcome": "ok" | "cancelled" | "error:<ExceptionName>",
         "started_at": "...",
         "ended_at":   "..." }
```

**Part B**, subscribe to these on the SSE channel that `ScenarioStrip`
already uses. `RanchMap`'s breadcrumb overlay can show `SCENARIO: coyote · 0:32`
by diffing `started_at` against `Date.now()`. Clear the breadcrumb on
`scenario.ended`. The `pass_idx` lets the UI differentiate the deterministic
seed=42 first pass from subsequent variety-seeded passes if needed.

## Rotation order (locked)

```
coyote → sick_cow → water_drop → storm → calving →
wildfire → rustling → cross_ranch_coyote
```

Matches `docs/ARCHITECTURE.md:121` hero loop. Pass 0 uses `seed=42`;
pass N uses `seed=42+N` for visual variety without touching the
byte-identical `make demo SEED=42 SCENARIO=all` path.

## Speed model

| `speed` | Wall time per scenario (`duration_s=600`) | 8-scenario cycle |
| ------- | ----------------------------------------- | ---------------- |
| 1       | ~10 min                                   | ~80 min          |
| **15** (default) | ~40 s                           | ~6 min           |
| 30 (video mode)  | ~20 s                           | ~2.5 min         |
| ≤ 0     | fast-path (no sleep)                       | ~3 s             |

Cooldown between scenarios: `max(1.0, 3.0 / max(speed, 0.1))` s — visible
idle-pause at 15x, negligible at 30x.

## Design notes worth keeping

- **Determinism preserved.** `_run_async` still delegates with `speed=0.0`
  and `assert_outcome=True`. `tests/test_determinism_e2e.py::
  test_demo_seed42_is_deterministic_3x` passes unchanged — proof that the
  refactor did not perturb event ordering or the random walk.
- **Ambient is a toggle, not a forced mutation.** `SKYHERD_AMBIENT=0` or
  `--no-ambient` gives back the pre-v1.1 empty-dashboard behaviour; useful
  for the field-demo laptop run where only Pi/Mavic events should appear.
- **Skip uses `asyncio.wait(FIRST_COMPLETED)`.** The skip-wait task races
  the play task; whichever finishes first wins, the other is cancelled
  cleanly. Guarantees `scenario.ended` always fires after `scenario.active`,
  even on exception (tested).
- **Broadcaster API is duck-typed.** Today's `EventBroadcaster._broadcast`
  is private; the plan's `emit`/`publish` is the public target. The driver
  tries `emit` → `publish` → `_broadcast` so no coupling to the current
  private API, and Part B (or a future broadcaster rename) needs no
  change here.
