<objective>
Produce and commit the fine-tuned MobileNetV3-Small classifier weights that Wave 3's pixel head loads.

Purpose: Wave 3 cannot build the pinkeye pixel head without a trained `.pth` file. Running training inside the head's `classify()` method at runtime blows the 500ms/frame budget (model load alone is 150-400ms) and introduces non-determinism. This plan isolates the training step: a deterministic one-off script generates ~500 synthetic frames from `renderer.py`, fine-tunes MobileNetV3-Small's head on them, saves `state_dict` to `src/skyherd/vision/_models/pinkeye_mbv3s.pth`, and verifies the weights pass a smoke test (positive frame yields non-healthy classification; negative frame yields healthy classification) before commit.

Output: `scripts/train_pinkeye_classifier.py` (dev-time only, excluded from coverage/pyright per existing patterns); `src/skyherd/vision/_models/pinkeye_mbv3s.pth` committed to git; `src/skyherd/vision/_models/WEIGHTS.md` documenting license + SHA-256.
</objective>

Label oracle (matches existing rule head severity tiers from src/skyherd/vision/heads/pinkeye.py):

```
def label_for_cow(ocular_discharge: float, has_pinkeye_flag: bool) -> int:
    if ocular_discharge <= 0.4 and not has_pinkeye_flag:
        return 0  # healthy
    if has_pinkeye_flag and ocular_discharge <= 0.4:
        return 2  # log (flag override)
    if ocular_discharge < 0.6:
        return 1  # watch
    if ocular_discharge < 0.8:
        return 2  # log
    return 3  # escalate
```

Eye-crop region (reverse-projection from renderer._draw_cow_blob — Wave 3's detector.py will replicate):

```
# Per cow at world pos (px, py) in bounds_m=(2000, 2000), frame 640x480:
fx = int(px / 2000.0 * 640)
fy = int((1.0 - py / 2000.0) * 480)
# Body ellipse: half-axes r_x=18, r_y=12.
# Head x: cx + r_x - 4.  Head y: cy - r_y + tilt - 6 where tilt = cow.lameness_score.
# Eye 48x48 crop window: centered on (hx, hy), half=24.
```

|----------|-------------|
| On-disk `.pth` weights file -> `torch.load` | Pickle-based format is a code-execution attack surface when untrusted |
| Training script -> renderer.py | Training script runs arbitrary PIL rendering code at dataset-gen time; trusted (local repo code only) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | T (Tampering) | pinkeye_mbv3s.pth (pickle format) | mitigate | Wave 3's pinkeye.py MUST use `torch.load(..., weights_only=True)` (torch 2.4+). This task documents SHA-256 in WEIGHTS.md; Wave 3 adds runtime hash verification |
| T-02-06 | T (Tampering) | Swapped .pth silently mislabels sick cows as healthy | mitigate | SHA-256 recorded in WEIGHTS.md; Wave 3 tests verify classifier differentiates positive from negative frames (smoke test on fixtures) |
