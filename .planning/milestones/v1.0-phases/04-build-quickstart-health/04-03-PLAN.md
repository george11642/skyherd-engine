<objective>
Fix BLD-02: prove the README 3-command judge quickstart actually works on a clean checkout. Today it's a promise in markdown with no enforcement — if anyone accidentally breaks the flow (lockfile desync, dep removal, Makefile typo), the first person to notice is the judge at 5pm on submission day.

Purpose: The hackathon judges will literally clone this repo and run 3 commands. If any step breaks, the submission is effectively disqualified regardless of how good the sim is. This plan wires a bash smoke script + CI job + README doc-drift test so the quickstart promise is asserted on every nightly run (and on-demand via `workflow_dispatch`).

Output:
- `scripts/fresh_clone_smoke.sh` — ~40-line bash script that creates a mktemp sandbox, `file://` clones the current checkout into it, runs `uv sync`, `pnpm install --frozen-lockfile && pnpm run build`, `make demo SEED=42 SCENARIO=all`, spawns `uvicorn skyherd.server.app:app` in background, polls `/health` until responsive, kills the server, and exits 0.
- `.github/workflows/fresh-clone-smoke.yml` — nightly + `workflow_dispatch` GH Actions job on ubuntu-latest with `setup-uv@v5 enable-cache: false` (cold path mandatory per RESEARCH.md pitfall #4).
- `tests/test_readme_quickstart.py` — unit test that greps README.md for canonical command strings so silent doc drift fails CI.

**Coordination:** This plan does NOT depend on Plan 02 (live-mode dashboard). The smoke script probes `uvicorn skyherd.server.app:app` directly on a custom port — a generic health probe decoupled from whether `make dashboard` is live or mock. That keeps this plan parallel with Plan 02 in Wave 2.
</objective>

```bash
#!/usr/bin/env bash
# fresh_clone_smoke.sh — verify the README 3-command quickstart on a clean checkout.
#
# Usage:
#   bash scripts/fresh_clone_smoke.sh
#
# Exits 0 if the full quickstart completes successfully; non-zero on any failure.
# Target runtime: < 5 min on a cold Ubuntu GitHub Actions runner.
#
# Environment variables:
#   GITHUB_WORKSPACE  — repo root to clone from (default: $(pwd))

set -euo pipefail

START=$(date +%s)
SANDBOX=$(mktemp -d -t skyherd-smoke.XXXXXX)
SERVER_PID=""

cleanup() {
    if [ -n "$SERVER_PID" ]; then
        kill "$SERVER_PID" 2>/dev/null || true
    fi
    rm -rf "$SANDBOX"
}
trap cleanup EXIT INT TERM

SOURCE_REPO="${GITHUB_WORKSPACE:-$(pwd)}"

echo "===> [smoke] sandbox: $SANDBOX"
echo "===> [smoke] source:  $SOURCE_REPO"

# Step 0: clone via file:// (simulates fresh clone without touching origin)
git clone --depth 1 "file://${SOURCE_REPO}" "$SANDBOX/repo"
cd "$SANDBOX/repo"

echo "===> [smoke] step 1: uv sync"
uv sync

echo "===> [smoke] step 2: pnpm install + build"
(cd web && pnpm install --frozen-lockfile && pnpm run build)

echo "===> [smoke] step 3: make demo SEED=42 SCENARIO=all"
timeout 180 make demo SEED=42 SCENARIO=all

echo "===> [smoke] step 4: dashboard /health probe (generic server — Plan 02 independent)"
# Use the stable generic server entry-point — decouples smoke from whether
# make dashboard is live or mock (Phase 4 Plan 02 flips that default).
uv run uvicorn skyherd.server.app:app --port 18765 --log-level warning &
SERVER_PID=$!

# Poll /health for up to 20s
for i in $(seq 1 20); do
    if curl -sf "http://127.0.0.1:18765/health" > /dev/null 2>&1; then
        echo "===> [smoke] /health OK after ${i}s"
        break
    fi
    sleep 1
    if [ "$i" -eq 20 ]; then
        echo "===> [smoke] FAIL: /health never responded"
        exit 1
    fi
done

# Bonus: probe /api/snapshot returns 200 (mock or live both serve it)
curl -sf "http://127.0.0.1:18765/api/snapshot" > /dev/null || {
    echo "===> [smoke] FAIL: /api/snapshot did not respond"
    exit 1
}

END=$(date +%s)
echo "===> [smoke] PASS in $((END - START))s"
```

```yaml
name: Fresh-clone smoke

# BLD-02: prove the README 3-command quickstart works on a clean judge
# environment. Runs nightly (caught before submission morning) and on
# manual dispatch (for pre-submission sanity checks). NOT on every PR —
# 5-min cold-path cost is too expensive for per-PR gating.

on:
  workflow_dispatch:
  schedule:
    # Nightly at 08:00 UTC (covers US + EU morning)
    - cron: "0 8 * * *"

jobs:
  fresh-clone-smoke:
    name: Fresh-clone smoke (<=5 min SLA)
    runs-on: ubuntu-latest
    timeout-minutes: 10

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Install uv
        uses: astral-sh/setup-uv@v5
        with:
          python-version: "3.12"
          # CRITICAL: no cache — must simulate true cold judge environment.
          # Research pitfall #4: caching masks the fresh-clone SLA.
          enable-cache: false

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: "20"

      - name: Setup pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 9

      - name: Assert worlds/*.yaml tracked in git (BLD-01 prerequisite)
        run: |
          test -f worlds/ranch_a.yaml
          test -f worlds/ranch_b.yaml

      - name: Run fresh-clone smoke
        run: bash scripts/fresh_clone_smoke.sh

      - name: Report timing on failure
        if: failure()
        run: |
          echo "Fresh-clone smoke FAILED. Inspect logs above for which step"
          echo "(uv sync / pnpm / make demo / /health) exceeded budget or errored."
```

```python
"""README must contain the canonical 3-command judge quickstart (BLD-02 doc-drift guard).

Locks the quickstart commands against silent edits. If anyone removes or
rewords the canonical strings in README.md, this test fails loudly.

The same commands also appear in CLAUDE.md and scripts/fresh_clone_smoke.sh —
all three must agree.
"""

from __future__ import annotations

from pathlib import Path

_REPO_ROOT = Path(__file__).parent.parent
_README = (_REPO_ROOT / "README.md").read_text(encoding="utf-8")
_CLAUDE_MD_PATH = _REPO_ROOT / "CLAUDE.md"

_EXPECTED_COMMANDS = [
    "uv sync",
    "pnpm install",
    "pnpm run build",
    "make demo SEED=42 SCENARIO=all",
    "make dashboard",
]


def test_readme_quickstart_commands_present() -> None:
    """README must contain each canonical quickstart command string verbatim."""
    for cmd in _EXPECTED_COMMANDS:
        assert cmd in _README, (
            f"README missing quickstart command: {cmd!r}. "
            "If you intentionally reworded the quickstart, update "
            "_EXPECTED_COMMANDS in this test AND scripts/fresh_clone_smoke.sh."
        )


def test_readme_has_quickstart_section() -> None:
    """README must have a Quickstart section (case-insensitive)."""
    assert "Quickstart" in _README or "quickstart" in _README.lower(), (
        "README is missing a Quickstart section — judges will not find the "
        "3-command flow."
    )


def test_claude_md_agrees_on_demo_command() -> None:
    """CLAUDE.md must reference the same make demo command as README."""
    if not _CLAUDE_MD_PATH.exists():
        # CLAUDE.md is not always present in every repo layout — skip gracefully
        return
    claude_md = _CLAUDE_MD_PATH.read_text(encoding="utf-8")
    assert "make demo SEED=42 SCENARIO=all" in claude_md, (
        "CLAUDE.md has drifted: it no longer references the canonical "
        "'make demo SEED=42 SCENARIO=all' command. Update CLAUDE.md to match "
        "README.md or update the README if the canonical command changed."
    )
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `scripts/fresh_clone_smoke.sh` → mktemp sandbox | `rm -rf "$SANDBOX"` is always scoped to a freshly-mktemp'd dir |
| GitHub Actions runner → repo code | Standard PR/push trust model; fresh-clone adds no new surface |
| README.md edits → test failure | Doc-drift test asserts canonical commands; editors cannot silently break the quickstart promise |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08 | Denial of Service | Smoke script `rm -rf "$SANDBOX"` accidentally removing non-sandbox dir | mitigate | Use `mktemp -d` (always yields fresh dir) + `trap cleanup EXIT INT TERM`; never use env var-expanded paths in `rm -rf` without the sandbox prefix check. Quote the path in `rm -rf "$SANDBOX"`. |
| T-04-09 | Denial of Service | Dashboard hangs the CI job indefinitely | mitigate | Spawn uvicorn with `&` in background, poll `/health` with 20s cap, kill PID in cleanup trap, `timeout-minutes: 10` on the workflow job. (Research pitfall #5.) |
| T-04-10 | Tampering | Malicious PR modifies `scripts/fresh_clone_smoke.sh` to exfiltrate secrets via CI | accept | PR-based CI already executes arbitrary repo code; fresh-clone smoke is no broader than existing `test` / `lint` jobs. Standard GH Actions trust model applies. |
| T-04-11 | Information Disclosure | Doc-drift test exposes CLAUDE.md content in failure messages | accept | CLAUDE.md is committed to the repo, not secret. Failure message only quotes the command string being searched for. |
</threat_model>
