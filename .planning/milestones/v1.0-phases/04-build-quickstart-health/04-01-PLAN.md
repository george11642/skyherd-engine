    def _default_world_config() -> Path:
        """Return the packaged ``worlds/ranch_a.yaml`` as a filesystem Path.

        Uses ``importlib.resources.files()`` so this works in both editable
        installs (finds repo-root ``worlds/`` via the force-include layout)
        and wheel installs (finds ``skyherd/worlds/`` inside the package).

        CRITICAL: do NOT use ``with as_file(p) as path: return path`` — the
        context manager exits before the caller uses the path, breaking on
        zipimport installs. The ``Path(str(traversable))`` cast is safe for
        uv-managed venv installs (SkyHerd's only supported install mode).
        """
        traversable = files("skyherd").joinpath("worlds/ranch_a.yaml")
        try:
            return Path(str(traversable))
        except (TypeError, ValueError):
            # Rare fallback: zipimport. Extract to a tempfile.
            import tempfile
            tmp = Path(tempfile.mkstemp(suffix="_ranch_a.yaml")[1])
            tmp.write_bytes(traversable.read_bytes())

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Wheel build → user install | `force-include` could leak unintended files if mapping is too broad |
| Runtime → filesystem | `importlib.resources` resolves to a real path; tempdir fallback triggers on zipimport |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | `pyproject.toml` force-include mapping | mitigate | Narrow mapping to exactly `"worlds" = "skyherd/worlds"` — do NOT use broad patterns like `"." = "skyherd"` which would ship `.env.local`, `runtime/*.db`, `.venv/`. Verify with `unzip -l dist/*.whl \| grep -vE "^(skyherd/\|\\.dist-info)"` returns no unexpected files. |
| T-04-02 | Denial of Service | `_default_world_config()` tempdir fallback | accept | Fallback only triggers on zipimport (not SkyHerd's install model); tempfile never deleted by helper, but OS /tmp GC handles it. Worst case: one leaked 2KB file per cold boot — negligible. |
| T-04-03 | Tampering | `importlib.resources.files("skyherd")` | accept | Requires filesystem write access to site-packages — equivalent to any other package tampering threat, not phase-specific. Standard PyPA trust model applies. |
</threat_model>
