# Requirements: SkyHerd Engine — MVP Completion Milestone

**Defined:** 2026-04-22
**Core Value:** The 3-minute demo video must land "oh damn" inside the first 30 seconds on a pure-sim run, deterministically, every replay. Sim perfection beats hardware novelty every time a battery dies mid-demo.

## v1 Requirements

### Managed Agents Persistence (the $5k Managed Agents prize architecture)

- [x] **MA-01**: Each of the 5 agents (FenceLineDispatcher / HerdHealthWatcher / PredatorPatternLearner / GrazingOptimizer / CalvingWatch) runs on exactly ONE long-lived platform session that persists across events within a scenario run. Zero fresh `SessionManager()` instantiations per event.
- [x] **MA-02**: `src/skyherd/scenarios/base.py::_DemoMesh` holds a session registry keyed by agent name; dispatch reuses the existing session or creates it lazily on first wake.
- [x] **MA-03**: Coyote scenario end-to-end creates at most 5 platform sessions total (one per agent) vs current 241. Verified by `SessionManager` instantiation count in test harness.
- [x] **MA-04**: Cost ticker shows idle-pause behavior: when no events are routed to an agent for N seconds, its `rate_per_hr_usd` drops to 0.0 and `all_idle: True` appears on the SSE stream.
- [x] **MA-05**: Checkpoint persistence works — `PredatorPatternLearner` running nightly keeps context across sim-day boundaries within a scenario run.

### Agent Routing Correctness

- [x] **ROUT-01**: `PredatorPatternLearner` is present in `_registry` dict in `scenarios/base.py`.
- [x] **ROUT-02**: Routing table in `scenarios/base.py` includes entries for `thermal.anomaly` → `[FenceLineDispatcher, PredatorPatternLearner]` and `skyherd/+/cron/nightly` → `[PredatorPatternLearner]`.
- [x] **ROUT-03**: Rustling scenario assertions verify agent dispatch count (not just event presence in the stream).
- [x] **ROUT-04**: All 5 agents have at least one assertion in at least one scenario that proves they actually ran (`AgentDispatched` counter ≥ 1 per agent across the scenario suite).

### Vision Credibility (the Keep Thinking $5k pattern-match)

- [x] **VIS-01**: At least one disease head (target: pinkeye — most visually obvious for a judge-facing demo) is rewritten to perform real pixel-level inference on the rendered PNG frame. Not threshold classification on `Cow.ocular_discharge`.
- [x] **VIS-02**: The pixel-head uses a MIT/BSD-licensed backbone (e.g. MegaDetector V6 as animal detector + small task-specific head, or a distilled CNN on rendered frames). No Ultralytics / AGPL imports.
- [x] **VIS-03**: The pixel-head and the remaining 6 rule-based heads share the same `DiseaseHead` ABC interface; pipeline output format unchanged.
- [x] **VIS-04**: Pixel-head inference runs on synthetic frames in under 500ms per frame on CPU (dev-box baseline) — cannot slow the sim below 2× real time.
- [x] **VIS-05**: One scenario (e.g. sick_cow) has a visible dashboard panel showing the pixel-head's detection with bounding box + confidence — real, not faked.

### Build & Quickstart Health

- [x] **BLD-01**: `make_world()` accepts default `config_path` resolving to `worlds/ranch_a.yaml` via `importlib.resources` or package-relative `Path`. Judge quickstart `make_world(seed=42)` works without arguments.
- [x] **BLD-02**: Fresh-clone flow documented in README verified on a second machine (or clean worktree): `git clone ... && cd skyherd-engine && uv sync && make demo SEED=42 SCENARIO=all` completes in under 5 minutes and all scenarios PASS.
- [x] **BLD-03**: `make dashboard` in live (non-mock) mode serves a functional dashboard from a fresh clone.
- [x] **BLD-04**: SITL smoke test added to CI using `SITL_IMAGE=ardupilot/ardupilot-sitl:Copter-4.5.7` pre-built image; single smoke scenario proves real MAVLink mission upload + arm + takeoff + RTL in under 2 minutes.

### Dashboard Live-Mode & Polish

- [x] **DASH-01**: `make dashboard` (without `SKYHERD_MOCK=1`) injects real `mesh`, `world`, and `ledger` into the FastAPI app; `/api/snapshot` returns real sim data.
- [x] **DASH-02**: Server live-path coverage increases to ≥85% (from current 73%).
- [x] **DASH-03**: Cost ticker UI shows visible "paused" state when `all_idle: True`; tested on camera in a golden-path scenario run.
- [x] **DASH-04**: Attestation panel streams live Merkle chain appends via SSE; verify-chain button in UI.
- [x] **DASH-05**: Motion polish pass — agent-lane entry animations, ranch map drone trail fade, predator pulse ring smoothing. No design-system rebuild.
- [x] **DASH-06**: Dashboard demonstrates all 5 agents on distinct agent lanes with real session IDs, real wake events, real tool calls (not mock-generated). Lighthouse score ≥90 on the SPA.

### Scenario Completeness

- [x] **SCEN-01**: Sick-cow scenario produces a rendered, rancher-readable vet-intake packet (drafted by HerdHealthWatcher → Doc skill or direct) and logs it to a retrievable location. Asserted in scenario test.
- [x] **SCEN-02**: All 8 scenarios (coyote / sick_cow / water_drop / calving / storm / cross_ranch_coyote / wildfire / rustling) continue to PASS after all other changes. Zero regression.
- [x] **SCEN-03**: Deterministic replay strengthened — scenario JSONL output byte-identical at hash level after timestamp sanitization; `make sim SEED=42` verified stable across three back-to-back runs.

### Code Hygiene

- [x] **HYG-01**: Replace the 15+ identified `except: pass` blocks (listed in `CONCERNS.md §3`) with `except Exception as exc: logger.warning(...)`. No bare silent-catch remains in `src/skyherd/`.
- [x] **HYG-02**: Twilio auth token env var standardized on `TWILIO_AUTH_TOKEN` across `voice/call.py` and `mcp/rancher_mcp.py`; `.env.example` updated; deprecation warning on `TWILIO_TOKEN`.
- [x] **HYG-03**: `agents/cost.py` coverage raised to ≥90% with tests for idle-pause billing, active-state delta accumulation, and the `all_idle` aggregation path.
- [x] **HYG-04**: Ruff + pyright run clean. The 15 pyright errors in `drone/pymavlink_backend.py` / `drone/sitl_emulator.py` resolved or added to typed-ignore list with rationale comments.
- [x] **HYG-05**: Test coverage gate remains 80%+; actual project-wide coverage holds or exceeds 87%.
