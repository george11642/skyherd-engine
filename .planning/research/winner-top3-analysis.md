## Cross-cutting patterns (top 3 only)

These are the techniques that appear in **at least 2 of the 3** Gemini analyses. Each is a hard signal — the prior synthesis that wasn't grounded in real video can be checked against this list.

1. **Three-act 3:00 lockstep.** All three videos hit ~03:00 exactly with a clean Setup / Demo / Close split. CrossBeam: 1:07 / 2:32 boundaries. Elisa: 0:10 / 1:47. PostVisit: 1:06 / 2:42. **Setup runs ~1/3, demo ~1/2, close ~1/6.** This is not coincidence — it is the dominant 4.6 winner skeleton.

2. **Founder credibility before product.** All three open with founder context (lawyer / dad-engineer / cardiologist) before showing UI. None lead with a product screenshot. The credibility arc is: who I am → why this hurts → how I built it.

3. **Domain-anchored hook, not metric hook.** **Zero of the three put a before/after numerical metric in the first 15 seconds.** CrossBeam uses a contrarian frame ("not a housing crisis, a permit crisis"). Elisa uses a personal story ("My daughter is in 7th grade"). PostVisit uses identity ("Hi, I'm Michał. I'm a cardiologist."). The hook anchor is human/domain truth, not a number.

4. **Hard cuts dominate.** All three use ~95% hard cuts. CrossBeam adds smooth canvas pans only during the architecture diagram. PostVisit adds dissolves only into title cards. **No whip-pans, glitches, or fancy transitions.** Cleanliness > flashiness.

5. **Centered composition everywhere.** Talking heads, UI shots, title cards — all dead-center. Easy to track, predictable, professional. None use rule-of-thirds for the founder shot.

6. **Studio-mic VO + lower-quality testimonial mic is acceptable.** CrossBeam and PostVisit both have studio-grade narration paired with camera-mic / Zoom-mic testimonials. The contrast is not punished — it actually adds authenticity.

7. **The "under the hood" reveal is a recurring depth moment.** CrossBeam's Opus Orchestrator node diagram (1:25-1:50). Elisa's Mission Control task graph (1:05) + GitHub repo scroll (2:02). PostVisit's API & Skills slide (2:49). **All three explicitly visualize the engineering stack mid-to-late video to satisfy the "depth" judging criterion.**

## Things to avoid (anti-patterns from top 3)

1. **No neon / cyberpunk / dark-mode futurism.** All three winners use natural, warm, or sterile-medical color grades. A glowing-purple AI dashboard look will read as 2022 ChatGPT-wrapper aesthetic. Match the domain (rangeland = warm earth tones, dawn light, dusty saturation).

2. **No opening-with-a-metric hook.** Zero winners did this. A hook that starts "Ranchers lose $4.2B annually to predators" will underperform a contrarian/identity hook ("Everyone thinks ranches need more sensors. They need a nervous system.").

3. **No fancy transitions.** Zero whip-pans, zero glitch transitions, zero morph-cuts in the top 3. Hard cuts and the occasional smooth canvas pan. Anything else reads as a 2024 YouTube-bro aesthetic.

4. **No mid-section >5 seconds without VO.** PostVisit's [1:35-1:49] sag was Gemini's flagged weak spot. Elisa's [00:25-01:00] block-snapping sag was the other. **Keep narration continuous through the demo segment** — an idle 5-second screencast will lose judges.

5. **No always-on captions of generic VO.** Only PostVisit used always-on captions and they were minimal. CrossBeam and Elisa skipped them entirely. Reserve subtitles for testimonials, lower-thirds, and emphasis kinetic-typography moments only — don't paste the whole VO on screen.

6. **No "Opus 4.6 wrote this whole app" boast as the depth moment.** Elisa flirted with this and Gemini flagged the [00:25-01:00] section as a sag. The depth moment must be **engineering specifics** (test count, architecture diagram, attestation chain) — not "AI built it for me."
