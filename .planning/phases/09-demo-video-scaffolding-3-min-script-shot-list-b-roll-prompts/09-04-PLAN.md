## Tasks

### Task 1 — Build `tests/hardware/test_preflight_e2e.py` (type="auto")

Structure:
```python
"""Phase 9 PF-04: end-to-end Friday workflow simulation.

All external IO is mocked. Exercises the exact sequence George will run on
Friday morning:

1. Two Pis publish edge_status heartbeats to an in-memory MQTT mock.
2. /api/edges aggregates and reports both online.
3. Pi-A publishes camera.motion (coyote).
4. FenceLineDispatcher (mocked via LocalSessionManager) emits a drone mission.
5. Mavic adapter (stub backend) accepts the mission and echoes telemetry.
6. Attestation ledger records the dispatch event.

Runs in <30s, no Docker, no real hardware.
"""
```

Key fixtures:
- In-memory MQTT broker (use `amqtt` test harness or stub `aiomqtt.Client`).
- Fake Pi heartbeat generator (reuses `src/skyherd/edge/*` data classes).
- `DRONE_BACKEND=stub` env for the Mavic adapter.
- `SKYHERD_AGENTS=local` to force LocalSessionManager (deterministic, no API).
- FreezeGun for deterministic timestamps.

Assertions:
- Both `edge-house` and `edge-barn` present in `/api/edges` response within
  90s simulated time.
- `camera.motion` event produces a `mavic.mission` event on the bus within
  5s simulated time.
- Attestation event list includes `fence.breach`, `drone.dispatched`, and
  `mission.completed`.
- Total wall time for test: < 30s.
