# Plan 09-02 — `make rehearsal` + `make record-ready` Targets

## Objective

Give George two Makefile commands he can slap on the terminal before recording:

1. `make rehearsal` — loops the sim indefinitely so George can practice
   voiceover without babysitting the terminal. Deterministic, zero config.
2. `make record-ready` — one-shot preflight that silences notifications,
   pre-warms the dashboard at :8000, prints the scrub-points from the script,
   and exits leaving the dashboard running.

## Success criteria

- [ ] `make rehearsal` target added; repeats `make demo SEED=42 SCENARIO=all`
  in a loop (Ctrl-C to stop). Respects `SEED` override.
- [ ] `make record-ready` target added; performs preflight steps and prints a
  single-line OK banner on success.
- [ ] Both targets are `.PHONY` and documented via `##` comments so
  `make help` (if exists) discovers them.
- [ ] Unit test `tests/test_makefile_record_targets.py` (or extend existing)
  proves both targets exist and parse cleanly via `make -n`.
- [ ] No changes to existing targets' behavior.

## Tasks

### Task 1 — Add `make rehearsal` target (type="auto")

```makefile
rehearsal:  ## Loop sim SEED=42 SCENARIO=all until Ctrl-C (for voiceover practice)
	@echo "Rehearsal loop — press Ctrl-C to stop. Seed=$(SEED)."
	@while true; do \
		$(MAKE) demo SEED=$(SEED) SCENARIO=$(SCENARIO) || break; \
		echo "--- loop: restarting in 2s ---"; \
		sleep 2; \
	done
```

Respect `SEED` (default 42) and `SCENARIO` (default all).

### Task 2 — Add `make record-ready` target (type="auto")

```makefile
record-ready:  ## Pre-shoot preflight: dashboard + scrub-points + readiness banner
	@echo "=== SkyHerd record-ready preflight ==="
	@echo "[1/4] Checking dashboard build artifacts..."
	@test -f web/dist/index.html || (echo "Dashboard not built — running pnpm build"; \
		cd web && (pnpm install --frozen-lockfile || pnpm install) && pnpm run build)
	@echo "[2/4] Seeding demo determinism check..."
	@uv run skyherd-demo play coyote --seed $(SEED) >/dev/null 2>&1 \
		&& echo "    OK — coyote scenario runs" || echo "    FAIL — coyote scenario broken"
	@echo "[3/4] Printing scrub-points from DEMO_VIDEO_SCRIPT.md..."
	@grep -E '^(### |- \*\*[0-9]+:[0-9]+)' docs/DEMO_VIDEO_SCRIPT.md | head -20 || true
	@echo "[4/4] Launching dashboard at http://localhost:8000"
	@echo ""
	@echo "=== READY TO RECORD ==="
	@echo "Open http://localhost:8000 — silence system notifications now."
	@uv run python -m skyherd.server.live --port 8000 --host 127.0.0.1 --seed $(SEED)
```

Notes:
- Uses `uv run skyherd-demo play coyote --seed` as a lightweight sanity check
  (single scenario, <5s) rather than full SCENARIO=all (too slow for preflight).
- Scrub-points grep pulls `### 0:25` style section headings and `- **0:40**`
  style bullet-anchors from the new script.
- Final step launches the live dashboard in the foreground, which George
  keeps running while recording.
