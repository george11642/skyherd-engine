# Plan 09-03 — Bootstrap.sh Dry-Run + Pi-Fleet Runbook Update + Pre-Flight Checklist

## Objective

Guarantee Friday's plug-in works first try. Three deliverables:

1. Dry-run audit of `hardware/pi/bootstrap.sh`: confirm idempotent, no interactive
   prompts, works on any Ubuntu-24.04-family host.
2. Update `docs/HARDWARE_PI_FLEET.md` with an **exact 15-minute Friday sequence**
   using George's actual hardware (2 × Pi 4 + Mavic Air 2, collar-free).
3. Create `docs/PREFLIGHT_CHECKLIST.md` — 20-item checklist each with verify
   command + expected output.

- [ ] `bash hardware/pi/bootstrap.sh --dry-run` with a valid creds file exits 0

  audit confirms).

- [ ] Any issues discovered → fixed in-place (deviation Rule 2/3).

  with exact copy-paste one-liners for Pi-A, Pi-B, and Mavic pairing.

  `verify:` command + expected output.

## Tasks

### Task 1 — Audit `hardware/pi/bootstrap.sh` idempotency (type="auto")

Steps:
1. Run `bash -n hardware/pi/bootstrap.sh` — must exit 0 (syntax).
2. Run twice back-to-back with `SKYHERD_CREDS_FILE=tests/hardware/fixtures/creds_good.json --dry-run` — both must exit 0 with identical stdout.
3. Grep for interactive commands: `read`, `sudo -S`, `prompt`, `confirm`. Any
   interactive path (other than wifi reconfigure, which is gated on missing
   default route) is a defect — fix via deviation Rule 3.
4. Confirm all `apt-get install` invocations use `-y`.
5. Confirm `credentials.example.json` is a valid JSON file.

Document audit findings inline in a new `## Idempotency Audit` block in
`docs/HARDWARE_PI_FLEET.md` (see Task 2).
