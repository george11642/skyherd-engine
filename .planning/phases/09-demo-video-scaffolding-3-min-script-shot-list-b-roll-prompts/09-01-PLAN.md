# Plan 09-01 — Video Scaffolding: Script, Shots, Submission, Socials

## Objective

Produce the five submission-ready documents George will reference Fri–Sun while
recording, editing, and submitting the Devpost video. All five are markdown, all
five are skimmable, all five pair with the deterministic `make demo SEED=42
SCENARIO=all` sim so scenes can be paused/scrubbed to exact moments.

## Success criteria

- [ ] `docs/DEMO_VIDEO_SCRIPT.md` exists, ≥ 150 lines, shot-by-shot with
  timecodes + sim-scrub anchors.
- [ ] `docs/SHOT_LIST.md` exists, lists every screen capture + physical +
  B-roll shot, includes ≥ 3 image-gen prompts for stylized overlays.
- [ ] `docs/SUBMISSION.md` exists with 100–200 word summary (word count verified
  in body), repo URL, YouTube URL placeholder, 3 category selections.
- [ ] `docs/LINKEDIN_LAUNCH.md` exists with hook + body + CTA and an
  `[APPROVE_BEFORE_POST]` marker near the top.
- [ ] `docs/YOUTUBE.md` exists with title (≤100 chars), description with
  timestamp index, tags, thumbnail brief.
- [ ] Existing `docs/VIDEO_SCRIPT.md` is NOT deleted; new script is a companion
  tightened for sim-first recording.

## Tasks

### Task 1 — `docs/DEMO_VIDEO_SCRIPT.md` (type="auto")

Shot-by-shot sim-keyed script with:
- 3-act structure (hook / demo / close), 3:00 total.
- Each beat annotated with the exact sim scenario trigger (`scenario=coyote`,
  `scenario=sick_cow`, etc.) and the wall-clock point at which to pause.
- Voiceover lines (Wes cowboy persona cues separated from George VO).
- Screen-capture markers noting which panel to grab.
- "If you only film one thing" fallback ordering.

### Task 2 — `docs/SHOT_LIST.md` (type="auto")

- Table of every shot with columns: `ID | Type | Duration | Subject | Notes`.
- Types: `screen-capture`, `physical-hero`, `b-roll`, `overlay`, `title-card`.
- Three image-gen prompts for stylized overlays (wireframe ranch, glowing
  sensor nodes, attestation chain visual) at the end.
- Cross-reference to `DEMO_VIDEO_SCRIPT.md` beats.

### Task 3 — `docs/SUBMISSION.md` (type="auto")

- 100–200 word summary (pulled/tightened from `docs/ONE_PAGER.md`).
- Word count comment in-line: `<!-- word count: 187 -->`.

- YouTube URL placeholder: `{{YOUTUBE_URL}}`.
- Category selections with rationale for each:
  - Best Use of Claude Managed Agents ($5k) — 5-agent mesh, idle-pause billing.

### Task 4 — `docs/LINKEDIN_LAUNCH.md` (type="auto")

- Hook (1-2 lines).
- Body (3-4 short paragraphs).

### Task 5 — `docs/YOUTUBE.md` (type="auto")
