# Plan 04-03 — API Pair Endpoint + SPA Viewer Route + Verified Badge

## Objective

Surface the Year-2 attestation story to judges:

1. **`GET /api/attest/by-hash/{hash}`** — one entry + prev-chain back to genesis (list of entries seq 1..N where N is the row with `event_hash=hash`).
2. **`GET /api/attest/pair/{memver_id}`** — the two-independent-receipts pairing: `{ledger_entry: {...}, memver: {id, sha256, path, agent}}`. Returns 404 if no ledger row has this memver_id.
3. **`/attest/:hash` SPA route** — new `AttestChainViewer` component rendering the chain walk with per-row verify status (PASS/FAIL badge).
4. **`AttestationPanel` per-row "Verified ✓" badge** — when user clicks Verify (existing), update each row with its verify result; clicking a row links to `/attest/:hash`.

## Tasks

1. **[auto]** Add `GET /api/attest/by-hash/{hash}` to `src/skyherd/server/app.py`. Mock mode: return a synthetic 3-row chain ending at the requested hash. Live mode: iterate ledger until matching hash, collect prev chain by walking `event_hash → prev_hash` backwards.
2. **[auto]** Add `GET /api/attest/pair/{memver_id}` to `src/skyherd/server/app.py`. Mock mode: return `{ledger_entry: _mock_attest_entry(), memver: {...}}`. Live mode: scan ledger for row where `payload_json` contains `"memory_version_id": "<memver_id>"` OR `_memver_id` field equals `memver_id`; pair with memory content from `get_memory_store_manager().read_by_version(...)` if available.
3. **[auto]** Add server tests `tests/server/test_attest_api.py` covering both endpoints in mock + live paths.
4. **[auto]** Create `web/src/components/AttestChainViewer.tsx` — fetches `/api/attest/by-hash/{hash}` + calls `/api/attest/verify`, renders chain with per-row PASS/FAIL chip. Uses existing `HashChip` + chip styles from `AttestationPanel`.
5. **[auto]** Register the route in `web/src/routes.tsx`: `if (path.startsWith("/attest/")) return <AttestChainViewer hash={...} />`.
6. **[auto]** Extend `web/src/components/AttestationPanel.tsx`:
   - Track per-row verify status after `Verify` completes (using the chain walk — if `first_bad_seq` set, all rows `seq < first_bad_seq` are VALID).
   - Render a small "✓" chip in the HASH column when verified.
   - Make each row's hash chip click-through to `/attest/{hash}` (via history.pushState).
7. **[auto]** Add vitest `web/src/components/AttestChainViewer.test.tsx` — renders chain, shows verify badge.
8. **[auto]** Smoke `AttestationPanel.test.tsx` still passes (extend with a verified-badge test).
