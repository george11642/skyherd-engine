# Plan 04-02 — skyherd-verify Standalone CLI

## Objective

Ship a **second** console-script `skyherd-verify` (separate from `skyherd-attest`) that does one thing: take a `{event.json, sig.hex, pubkey.pem}` triple, report PASS/FAIL with a human-readable trace, target <200ms wall time on a typical 10-event chain verification.

## Tasks

1. **[auto]** Create `src/skyherd/attest/verify_cli.py` with:
   - `verify-event` sub-command: verify a single `(event_json, sig_hex, pubkey_pem)` triple. Prints trace: `[PASS] hash match / sig valid` or `[FAIL] <reason>`. Exit 0 on PASS, 2 on FAIL.
   - `verify-chain` sub-command: point at a SQLite ledger DB, walk the chain, print pass/fail with per-step timing. Exit 0/2.
   - Uses existing `skyherd.attest.signer.verify` and `skyherd.attest.ledger.Ledger.verify()` — zero new deps.
2. **[auto]** Register console-script `skyherd-verify = "skyherd.attest.verify_cli:main"` in `pyproject.toml [project.scripts]`.
3. **[auto]** Add tests `tests/attest/test_verify_cli.py`:
   - `test_verify_event_pass` / `test_verify_event_fail_wrong_sig` / `test_verify_event_fail_bad_pubkey`
   - `test_verify_chain_pass` / `test_verify_chain_fail_tampered`
   - `test_verify_chain_under_200ms` (perf gate: 10-event chain < 200ms, p50 across 3 runs)

## Verification

- `uv run pytest tests/attest/test_verify_cli.py -q`
- `uv run pytest tests/attest/test_verify_cli.py --cov=src/skyherd/attest/verify_cli --cov-report=term-missing` — ≥90%.
- Manual: `uv run skyherd-verify verify-chain --db /tmp/attest.db` works end-to-end.

## Done criteria

- [ ] Console script wired.
