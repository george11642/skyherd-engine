# Plan 06-03 — docker-compose hardware-demo + Makefile + H2 runbook

## Objective

Package Phase 5 + Plan 06-01 + Plan 06-02 as a one-command laptop demo:

```bash
make hardware-demo-sim
```

boots (in one `docker compose up`):

- `mosquitto` — MQTT broker,
- `ardupilot-sitl` (Copter-4.5.7) — drone simulator,
- `skyherd-live` — FastAPI + SSE dashboard reachable at `http://localhost:8000`,
- `skyherd-edge-coyote` — runs `skyherd-edge coyote --seed 42` on a 2-s interval,
- `skyherd-pi-to-mission` — runs `skyherd-edge pi-to-mission --backend sitl`,
- `skyherd-speaker` — runs `skyherd-edge deterrent` (mute in CI).

The chain is: coyote harness → MQTT → pi-to-mission → SITL takeoff/patrol → speaker_bridge.

Also adds a fast `make h2-smoke` target that runs a non-docker unit-level smoke
of the full chain (<5 s).

## Context

- `docker-compose.sitl.yml` already exists — we add a **separate** laptop-local file
  so judges can run the hero demo without touching our CI stack.
- `Makefile` has `sitl-up`, `sitl-down`, `bus-up`, `bus-down` targets; we keep
  them unchanged and add new targets next to them.
- `docs/HARDWARE_DEMO_RUNBOOK.md` + `docs/HARDWARE_PI_FLEET.md` are the judge-facing
  docs; we add `docs/HARDWARE_H2_RUNBOOK.md` as the Phase-6-specific sibling.

## Implementation

### File 1 — `docker-compose.hardware-demo.yml`

```yaml
version: "3.9"

services:
  mosquitto:
    image: eclipse-mosquitto:2
    container_name: skyherd-mosquitto
    ports:
      - "1883:1883"
    command: >
      sh -c "echo 'listener 1883
      allow_anonymous true' > /mosquitto/config/mosquitto.conf
      && mosquitto -c /mosquitto/config/mosquitto.conf"

  sitl:
    image: ${SITL_IMAGE:-ardupilot/ardupilot-sitl:Copter-4.5.7}
    container_name: skyherd-sitl
    ports:
      - "14540:14540/udp"
      - "5760:5760/tcp"
    environment:
      SITL_HOME: "34.1230,-106.4560,1500,180"  # NM ranch_a SW fence
    healthcheck:
      test: ["CMD", "bash", "-c", "echo '' | nc -u -w1 localhost 14540 > /dev/null 2>&1 || exit 0"]
      interval: 5s
      timeout: 3s
      retries: 12

  skyherd-live:
    build:
      context: .
      dockerfile: docker/hardware-demo-edge.Dockerfile
    container_name: skyherd-live
    depends_on:
      - mosquitto
    ports:
      - "8000:8000"
    environment:
      MQTT_URL: "mqtt://mosquitto:1883"
      RANCH_ID: "ranch_a"
    command: ["uv", "run", "python", "-m", "skyherd.server.live", "--host", "0.0.0.0", "--port", "8000", "--seed", "42"]

  skyherd-edge-coyote:
    build:
      context: .
      dockerfile: docker/hardware-demo-edge.Dockerfile
    container_name: skyherd-edge-coyote
    depends_on:
      - mosquitto
    environment:
      MQTT_URL: "mqtt://mosquitto:1883"
      RANCH_ID: "ranch_a"
      COYOTE_CAM_ID: "coyote_cam"
    command: ["uv", "run", "skyherd-edge", "coyote", "--seed", "42", "--interval", "2.0"]

  skyherd-pi-to-mission:
    build:
      context: .
      dockerfile: docker/hardware-demo-edge.Dockerfile
    container_name: skyherd-pi-to-mission
    depends_on:
      - mosquitto
      - sitl
    environment:
      MQTT_URL: "mqtt://mosquitto:1883"
      RANCH_ID: "ranch_a"
      DRONE_BACKEND: "sitl"
      MAVSDK_HOST: "sitl"
      MAVSDK_UDP: "udp://:14540"
    command: ["uv", "run", "skyherd-edge", "pi-to-mission", "--backend", "sitl"]

  skyherd-speaker:
    build:
      context: .

    command: ["uv", "run", "skyherd-edge", "deterrent", "--mute"]
```

### File 2 — `docker/hardware-demo-edge.Dockerfile`

Minimal Python 3.11 + uv image that installs the repo wheel and exposes the
skyherd-edge + skyherd-live entry points.  Re-uses the repo source, does not
rebuild SITL.

```dockerfile
FROM python:3.11-slim

RUN apt-get update && apt-get install -y --no-install-recommends \
    curl build-essential ca-certificates \
    && rm -rf /var/lib/apt/lists/*

RUN curl -LsSf https://astral.sh/uv/install.sh | sh
ENV PATH="/root/.local/bin:${PATH}"

WORKDIR /skyherd
COPY pyproject.toml uv.lock README.md LICENSE ./
COPY src/ ./src/

RUN uv sync --frozen --no-dev

ENV PYTHONUNBUFFERED=1
EXPOSE 8000

CMD ["uv", "run", "skyherd-edge", "--help"]
```

### File 3 — `Makefile` additions

Insert after `hardware-demo` in the existing Makefile:

```make
hardware-demo-sim:  ## Laptop-local H2 demo: mosquitto + SITL + coyote + pi-to-mission + speaker
	@echo "Starting SkyHerd H2 laptop demo — coyote harness → SITL takeoff → deterrent"
	docker compose -f docker-compose.hardware-demo.yml up -d --build
	@echo "Dashboard: http://localhost:8000"
	@echo "Tail logs: docker compose -f docker-compose.hardware-demo.yml logs -f"

hardware-demo-sim-down:  ## Stop the H2 laptop demo stack
	docker compose -f docker-compose.hardware-demo.yml down

h2-smoke:  ## Fast (<5s) unit-level smoke of the H2 chain (no docker)
	uv run pytest tests/hardware/test_h2_e2e.py::TestH2Smoke -v --no-cov
```

### File 4 — `docs/HARDWARE_H2_RUNBOOK.md`

Judge-facing one-pager covering:

1. Summary — "Desk coyote → SITL takeoff in under 60 seconds."
2. Prereqs — Docker, 2 GB free, no Pi required.
3. One-shot boot:
   ```
   make hardware-demo-sim
   open http://localhost:8000
   ```
4. What you see — dashboard panels: incoming thermal frames, agent wake,
   drone state transitions (armed/in-air), attestation chain growing.
5. Shutdown — `make hardware-demo-sim-down`.
6. Troubleshooting grid (5 rows):
   - SITL port conflict → override with `SITL_PORT_UDP=14541`.
   - Image pull fails → fall back to `SITL_IMAGE=skyherd-sitl:local` + `make sitl-up`.
   - Dashboard 502 → check `docker compose logs skyherd-live`.
   - Coyote harness 0 messages → check `docker compose logs skyherd-edge-coyote | grep COYOTE_INTERVAL_S`.
   - "deterrent fired but silent" → expected (`SKYHERD_DETERRENT=mute`); unset and re-run on a laptop with audio.
7. Architecture diagram ASCII showing the 6 services + MQTT topic flow.
