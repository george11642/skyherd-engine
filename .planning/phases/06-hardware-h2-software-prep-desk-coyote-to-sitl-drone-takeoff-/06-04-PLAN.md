# Plan 06-04 — SITL failover (chaos-monkey) + full E2E test + coverage audit

## Objective

Two tests that close the Phase 6 acceptance loop:

1. **Chaos-monkey failover** — kill the SITL connection mid-mission, assert RTL
   is triggered and an attestation `sitl.failover` entry is written.  Uses a
   `FakeSITL` stub so tests stay offline, fast, deterministic.
2. **End-to-end integration** — cardboard_coyote emits thermal → `InMemoryBroker` →
   `PiToMissionBridge` → `FenceLineDispatcher` → drone mission → RTL →
   `Ledger.iter_events` confirms the full chain with seq 1..N, every prev_hash
   chains, every signature verifies.  <10 s wall.

Plus a small coverage-and-determinism audit to catch regressions before we ship.

## Context

- Plan 06-01 delivered `PiToMissionBridge` with RTL **not auto-triggered** (left
  hovering) — this plan adds explicit RTL-on-failure behaviour behind a
  `ChaosMonitor` helper.
- `StubBackend` is great for happy-path tests but doesn't simulate connection
  loss.  We add `tests/fixtures/fake_sitl.py` — a `FakeSITLBackend` that wraps
  `StubBackend` and lets tests inject a `DroneUnavailable` at a specified
  waypoint index.

### File 2 — `src/skyherd/edge/pi_to_mission.py` (amendment)

Add a `ChaosMonitor` helper and an explicit `failover()` method to the bridge:

```python
async def failover(self, reason: str) -> None:
    """Execute RTL and record an sitl.failover ledger entry."""
    try:
        await self._drone_backend.return_to_home()
        self._ledger_append("sitl.failover", {"reason": reason, "status": "rtl_ok"})
    except DroneError as exc:
        self._ledger_append("sitl.failover", {"reason": reason, "status": "rtl_failed", "error": str(exc)})
```

And wire it into `handle_event`: wrap the `backend.patrol()` call in a try/except
`DroneUnavailable` + `DroneTimeoutError` block; on exception call
`await self.failover(reason=str(exc))` **before** re-raising / returning.
