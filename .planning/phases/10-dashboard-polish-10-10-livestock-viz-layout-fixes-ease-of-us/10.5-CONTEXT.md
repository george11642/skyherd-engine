# Phase 10.5 Context — Final 10/10 Polish

## Why

Phase 10 landed the dashboard at 8.5/10. Two specific residual items were
called out in the Phase 10 SUMMARY "Subjective rating" section and confirmed
by the user as the remaining work to close to 10/10 before the Apr 26
submission:

1. **WebGL terrain tiles for the ranch map** — the current Canvas 2D
   background is a solid fill plus a diagonal linear gradient. Production
   dashboards (Palantir, Anduril) use stylized raster terrain that hints at
   landscape texture without being photorealistic. We want the same sage/dust
   design-token palette but with a noise-driven WebGL layer rendered beneath
   the existing 2D canvas.

2. **Inter-tick cow motion tweening** — the current cow dots snap to their
   new positions on each SSE `world.snapshot` tick (~1 Hz during scenarios).
   Smooth position interpolation with `requestAnimationFrame` and ease-out
   cubic over 800ms removes the choppy feel and makes the herd look alive.

## Non-goals

- NOT replacing the 2D canvas. The existing painter's-algorithm layer order
  (terrain → paddocks → fences → tanks → cows → drone → predators →
  overlays) stays intact. WebGL is purely a *background* layer mounted as
  a sibling `<canvas>` absolutely positioned beneath.
- NOT importing three.js / pixi.js / regl. Raw WebGL2 only, inline shader
  strings. Bundle budget (≤90 kB gzip) makes a library-free approach
  mandatory.
- NOT animating predator rings, drone triangle, weather overlay, or any
  other existing element. This is purely background terrain + cow position
  interpolation.

## Determinism

The simulator is 100% server-side Python. Both effects are pure client-side
visual polish that derive from the SSE snapshot stream. `make demo SEED=42
SCENARIO=all` is unaffected by construction — no files in
`src/skyherd/` are touched.

## Accessibility

`prefers-reduced-motion: reduce` MUST disable:
- Terrain noise animation (static shader evaluation at `t=0` fallback).
- Cow motion tweening (snap-to-target on each SSE update, old behavior).

The predator pulse ring already respects this rule — we match its pattern.

## Performance targets

- WebGL: single fullscreen-triangle draw call per frame, 3-octave value
  noise in fragment shader. < 3% CPU on M-series equivalent hardware.
- Cow tweening: O(cows) per RAF frame (~20-50 cows max), simple lerp math.
  Tiny compared to the existing 2D draw cost.
