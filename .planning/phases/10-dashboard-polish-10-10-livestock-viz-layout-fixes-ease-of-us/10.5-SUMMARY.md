# Phase 10.5 Plan: Final 10/10 Polish Summary

## One-liner

WebGL2 stylized terrain layer + single shared RAF tween pipeline bring
cows, drone, trail, and predators to life with ease-out-cubic motion
between SSE ticks — closing the 8.5 -> 10/10 dashboard gap under budget
with zero new runtime deps.

## What changed

### New: `web/src/lib/tween.ts` (+ `.test.ts`)

Pure math + immutable state helpers for the shared RAF pipeline:

- `easeOutCubic(t)` — `1 - (1-t)^3`, input-clamped.
- `lerp(a, b, t)` — re-exported as the single source of truth.
- `TweenState` (readonly): `{ start, target, tweenStartMs, durationMs }`.
- `createTween(start, target, nowMs, durationMs)` — seed a new tween.
- `tweenValue(state, nowMs)` — returns the current eased value;
  handles `durationMs <= 0` as instant snap.
- `retarget(state, newTarget, nowMs, newDurationMs?)` — immutable
  mid-flight retarget: new `start = current eased value`, preserving
  position continuity across SSE ticks.
- `lerpRgb(from, to, t, alpha)` — rounded-integer rgba string for
  color cross-fades.
- `prefersReducedMotion()` — safe `matchMedia` probe with SSR/jsdom
  fallback.

**25 vitest cases** cover all endpoints, monotonicity, retarget
continuity, clamping, and matchMedia edge cases.

### New: `web/src/components/shared/TerrainLayer.tsx` (+ `.test.tsx`)

Raw WebGL2 component, mounted absolutely behind the 2D canvas:

- Fullscreen-triangle vertex shader using `gl_VertexID` math (zero
  vertex buffers).
- Fragment shader: 3-octave fBm value noise + sage→dust gradient,
  darkened to bg-0 base, slow time-driven drift, radial vignette.
- Single `drawArrays(TRIANGLES, 0, 3)` per frame.
- DPR-aware — reads `getBoundingClientRect` + `devicePixelRatio`
  each frame and updates canvas backing size + `gl.viewport`.
- `prefers-reduced-motion: reduce` → pins `uTime` to 0 and exits
  the RAF loop after the first static frame. Responds to runtime
  preference changes via `matchMedia.addEventListener("change")`.
- Graceful 2D fallback: when `getContext("webgl2")` returns null OR
  shader compilation fails OR program linking fails, paints a
  sage/dust gradient via the 2D context so the map still has a
  backdrop. `paintFallback` is a pure 2D canvas draw — no WebGL
  dependencies.

**5 vitest cases** cover canvas render + aria-hidden, shader
compilation + program link + draw call, missing-WebGL2 fallback,
failed-shader fallback, and `prefers-reduced-motion` zero-time pin.

### Refactored: `web/src/components/RanchMap.tsx`

All moving entities now share one RAF pipeline:

- `EntityState`: per-id tween records for cows + predators, a
  nullable drone record, and a bounded trail point array with
  `spawnMs` for per-point fade-in.
- `applySnapshotToTweens(state, snap, nowMs, reduceMotion)` (exported
  for tests): on each SSE `world.snapshot`, new entities seed at their
  target with `fadeStartMs`; existing entities retarget via
  `retargetTween(...)` preserving continuity; health transitions
  trigger a 400ms color cross-fade from the previously-blended RGB
  to the new target RGB; the drone's trail appends a `{x, y,
  spawnMs}` point only on a meaningful target change, capped at
  `TRAIL_LEN=12`; entities absent from the new snapshot are culled.
- Per-entity tween durations (ms): cow position 800, drone position
  600, cow color 400, cow fade-in 500, predator fade-in 500, trail
  fade-in 400. `reduceMotion === true` forces durationMs=0 (snap).
- Single RAF loop for the 2D canvas reads `tweenValue(..., nowMs)`
  each frame; TerrainLayer runs its own separate WebGL RAF that
  exits early under reduced-motion.
- `<TerrainLayer />` mounted as first child inside the existing
  `<div className="relative w-full h-full">`; the 2D canvas keeps a
  translucent fill (0.45 day / 0.55 night) so the WebGL terrain
  reads through as a subtle base while paddock shading, labels, and
  entity overlays stay legible.
- `reduceMotionRef` stays in sync with `matchMedia` `"change"` events
  at runtime.

**9 new vitest cases** assert: new cow seeds at target; cow position
lerps through ease-out-cubic midpoint (0.725 at t=0.5 of 0.2→0.8);
reduced-motion snaps immediately; culled cows disappear; health
transitions trigger color cross-fade with correct from/to RGB;
drone tween seeds + retargets with 600ms duration; trail appends
only on target change; trail caps at 12 points; predators tween +
fade-in with `fadeStartMs`.

All 19 existing RanchMap tests (classifyCow, ScenarioBreadcrumb,
scenario zone glow, predator pulse ring motion) stay green after
refactor — the tween pipeline is purely additive where positions
flow from.

## Accessibility

`prefers-reduced-motion: reduce` disables every new animation:

- Terrain: `uTime` uniform pinned to 0; RAF loop exits after one
  static frame.
- Cow, drone, predator position tweens: durationMs forced to 0
  (snap to target on SSE tick).
- Cow color cross-fades: durationMs stays 0-effective (already
  jumps to target).
- Trail fade-in + entity fade-in: `Math.min(1, age/msBudget)`
  already resolves to ~1 immediately when the reduce-motion branch
  seeded the entity at its target with duration 0.

Runtime toggle support: both RanchMap and TerrainLayer listen to
`matchMedia("(prefers-reduced-motion: reduce)")` `"change"` events,
so flipping the OS preference mid-session takes effect on the next
frame without a reload.

## Performance

- TerrainLayer: single fullscreen-triangle draw per RAF frame.
  Fragment shader is 3 octaves of hash-based value noise (≈24 hash
  taps per pixel) — negligible on any GPU from the last 8 years. On
  reduced-motion, exits after the first frame so it's 0% CPU.
- Shared RAF: O(N_cows + N_predators + N_trail) pure number lerp per
  frame, plus the existing 2D canvas draw. ~20-50 cows + 0-3
  predators + ≤12 trail points → essentially free compared to the
  `ctx.arc/fill/stroke` cost already there. No measurable FPS change
  vs. Phase 10.

## Deviations from plan

**Scope expansion (user orchestrator mid-flight):** original plan
covered cow position tween + cow color fade only. Orchestrator
extended to drone, drone trail, and predators under one shared RAF.
Absorbed into the same `EntityState` + `applySnapshotToTweens`
abstraction with per-entity duration constants. No new deps, no
extra RAF loops, bundle well under budget.

No Rule 1/2/3 auto-fixes were needed — the existing codebase was
clean, and the refactor landed without touching any adjacent
components.
