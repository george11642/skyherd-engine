## Summary

Phase 5 is a **wiring and polish phase on top of an already-judged-worthy stack**. The audit verdict on `web/` is "HIGH QUALITY (judge-worthy)" — all 8 core components already exist (`RanchMap`, `AgentLane`, `AgentLanes`, `AttestationPanel`, `CostTicker`, `RancherPhone`, `CrossRanchView`). `framer-motion 12.38.0`, `react 19.1.0`, `tailwindcss 4.1.7`, `vite 6.3.5`, `@fontsource-variable/*` are all installed and in active use. `sse-starlette 2.4.1` + `fastapi 0.136.0` back the `/events` SSE stream, `pytest-asyncio 1.3.0` + `httpx 0.28.1` + `ASGITransport` are already the test pattern. The `CostTicker` component already renders a "PAUSED (idle)" chip with a muted state treatment — the visual contract exists, it just needs live-path data to fire it. The attestation ledger already has `Ledger.verify()` returning a `VerifyResult` model — no new crypto code; a button wires it to `/api/attest/verify`.

Four real gaps close DASH-01..06 + SCEN-01:
1. **Live-path injection** — `make dashboard` runs with `SKYHERD_MOCK=1`; need `make dashboard-live` target and a `SkyHerd.make_world()` → `AgentMesh` → `Ledger` wire-up in `server/app.py` lifespan (currently only accepts `mesh/world/ledger` as `create_app` kwargs).
2. **Live-path test coverage 73% → ≥85%** — `_real_cost_tick`, `_snapshot_loop` live branch, `_attest_loop` live branch, and `_live_agent_statuses` are the uncovered lines. `test_app_coverage.py` already has MagicMock-injected live fixtures (pattern exists); extend it.
3. **Vet-intake artifact** — sick-cow scenario produces `page_rancher` + `classify_pipeline` tool calls but emits no rancher-readable document. Need a markdown-format drafter on `HerdHealthWatcher` (or a new `draft_vet_intake` MCP tool) that writes to `runtime/vet_intake/A014_<ts>.md` and is linked via a new `/api/vet-intake/<id>` endpoint + `VetIntakePanel` modal.
4. **Polish + Lighthouse ≥90** — existing pulses/animations honour `prefers-reduced-motion`. `duration-240ms ease-out forwards` is the established idiom (`index.css:204`). Polish work: (a) drone trail fade already present in `RanchMap` (add alpha decay pass), (b) predator ring already has `threat-ring` keyframe but `RanchMap` draws a static ring — switch to CSS-driven overlay or RAF alpha interpolation, (c) font preloading in `index.html`, (d) code-split `RanchMap` Canvas work into a lazy chunk.

**Primary recommendation:** Do not introduce new libraries. Reuse `framer-motion`'s `useSpring/useTransform` (already in `CostTicker`), re-use the `index.css` keyframes (`pulse-dot`, `threat-ring`, `phone-ring`, `log-enter`, `fence-pulse`), and re-use the existing `AttestationPanel` / `CostTicker` / `AgentLane` shells — add one new `VetIntakePanel` and one new `verify-chain` button. The UI-SPEC (generated by `/gsd-ui-phase 5`) owns visual direction; this research locks the engineering surface.

### Pattern 1: FastAPI SSE with `EventSourceResponse` + backpressure

**What:** Each subscriber gets a bounded `asyncio.Queue(maxsize=100)`. Slow consumers drop oldest-first. SSE response includes `X-Accel-Buffering: no` to defeat intermediary buffering.

**When to use:** Current `EventBroadcaster` already does this correctly. Don't refactor — extend.

**Existing code (copy this shape for new event types):**
```python
# server/events.py:305-315 — already in repo
def _broadcast(self, event_type: str, payload: dict[str, Any]) -> None:
    for q in self._subscribers:
        try:
            q.put_nowait((event_type, payload))
        except asyncio.QueueFull:
            # Slow consumer — drop oldest, put newest
            try:
                q.get_nowait()
                q.put_nowait((event_type, payload))
            except (asyncio.QueueEmpty, asyncio.QueueFull):
                pass
```

**New event: `vet_intake.drafted` should reuse the same `_broadcast` call** with payload `{id, cow_tag, severity, path, ts}`. No new event loop.

**Sources:**
- `[CITED: github.com/sysid/sse-starlette]` — sse-starlette 2.4.1 is the library backing `EventSourceResponse`; production-ready, W3C-compliant SSE for Starlette/FastAPI
- `[CITED: fastapi.tiangolo.com/advanced/async-tests]` — `ASGITransport` + `AsyncClient` is the official test pattern; already used in `test_app.py`

### Pattern 3: framer-motion `useSpring` for animated numbers (already in use)

**What:** `CostTicker.tsx:69-91` already wraps the cumulative USD count with `useSpring({stiffness:60, damping:20})`. Any new animated number should use this exact shape.

**When to use:** Any value that changes continuously (cumulative cost, cumulative tokens). Do **not** use for discrete state flips (idle ↔ active) — those should be CSS keyframes or a simple class swap.

**Existing code:**
```typescript
// web/src/components/CostTicker.tsx:69-91 — already in repo
function AnimatedCost({ value }: { value: number }) {
  const spring = useSpring(value, { stiffness: 60, damping: 20 });
  const display = useTransform(spring, (v) => `$${v.toFixed(6)}`);
  useEffect(() => { spring.set(value); }, [value, spring]);
  return <motion.span style={...}>{display}</motion.span>;
}
```

**Sources:**

2. **Code-split Canvas renderer** (`vite.config.ts`):

3. **Set `<link rel="manifest">` early and `theme-color`** — PWA score contributes to Lighthouse PWA but not Performance; Performance is the ≥90 target.

4. **Avoid `console.log` in production** — already a global lint rule.

### Anti-Patterns to Avoid

- **Rebuilding the dashboard.** Audit flagged `web/` as "judge-worthy." Adding more components, adjusting color tokens, or renaming variables is churn.
- **Adding a charting library.** The sparkline is 20 lines of SVG. Recharts/visx are 200KB+ of context-free animation.
- **Introducing Zustand/Jotai.** Existing `useState` + SSE event handlers are sufficient for phase data flows.
- **Mock+live split via env var branching inside components.** Mock/live is a backend contract; the browser should see identical SSE shapes either way.
- **Migrating `framer-motion` → `motion`.** Rename-only; v12 under either name has identical React 19 support. Out-of-scope churn.
- **Testing SSE end-to-end with `AsyncClient.stream()`.** Known issue: hangs indefinitely with `ASGITransport` `[CITED: github.com/encode/httpx/issues/2186]`. Use direct `EventBroadcaster.subscribe()` (pattern in `test_app.py:141-158`).
- **Cost ticker "paused" = only a label change.** Must also change (a) border color, (b) cumulative number dimming, (c) agent-strip chip states, (d) sparkline fading. Judges watch video; single-label change is invisible at 3-min-demo distance.
- **Bundling `react-markdown` for the vet-intake modal.** 50KB for a fixed-schema document. Use the inline renderer.

## Common Pitfalls

### Pitfall 1: SSE stream buffering by nginx / proxies in production
**What goes wrong:** Events queue up for 16KB before browser sees them; dashboard "freezes" for 5-10 seconds then releases a burst.
**Why it happens:** Default nginx buffers `text/event-stream` unless told otherwise.
**How to avoid:** `sse-starlette.EventSourceResponse` already sets `Cache-Control: no-cache` but **does not** set `X-Accel-Buffering: no`. Since the `make dashboard` target runs uvicorn directly (no nginx), this is latent but catches us if a reverse proxy ever fronts the app. Add `X-Accel-Buffering: no` to `EventSourceResponse` headers proactively.
**Warning signs:** Event arrival in bursts rather than 1Hz steady stream. Browser DevTools → Network → `/events` → response time = infinite, "received" grows in chunks rather than smoothly.
`[CITED: github.com/sysid/sse-starlette]`

### Pitfall 2: `AsyncClient.stream()` + `ASGITransport` hangs in tests
**What goes wrong:** `test_events_sse_emits_via_http` style tests never complete; pytest hits timeout.
**Why it happens:** Documented httpx issue — ASGITransport does not stream responses; the test client wants to buffer the whole body.
**How to avoid:** Test the `EventBroadcaster` directly (as `test_app.py:141-158` already does — subscribe to an in-process instance). For HTTP-level testing, use a short-lived `uvicorn` subprocess + real TCP `httpx-sse` client.
**Warning signs:** Test timeout on any SSE HTTP route test; green EventBroadcaster unit tests.
`[CITED: github.com/encode/httpx/issues/2186]`

### Pitfall 3: `_real_cost_tick` reaches into private mesh internals
**What goes wrong:** `events.py:347-377` does `self._mesh._session_manager._tickers.get(session.id)` — private-attribute access. If Phase 1 rewires session storage, this silently returns `None` and the live cost tick is empty.
**Why it happens:** No public accessor. Tight coupling from broadcaster into mesh internals.
**How to avoid:** Add `AgentMesh.cost_snapshot() -> list[AgentCost]` as a public method (or wire it via a Phase 1 surface). Document in Phase 5 plan as an API-contract change on the Phase 1 boundary.
**Warning signs:** Live-mode dashboard shows cost ticker stuck at $0.00, agent states all "idle" — while scenarios actively fire in the server log.

### Pitfall 4: Lighthouse score regression from font preload mistakes
**What goes wrong:** Preloading 6 font weights (Fraunces 400/600/700, Inter 400/500, JetBrains 400) causes bandwidth contention; LCP gets WORSE.
**Why it happens:** Over-preloading competes with critical JS bundle download.
**How to avoid:** Preload only 2 — Fraunces 600 (display on RanchMap labels) + Inter 400 (body). Everything else loads normally. `[CITED: dev.to/leo_d/tackling-core-web-vitals-on-a-heavy-react-app]`
**Warning signs:** Lighthouse "Preload key requests" audit passes but FCP and LCP scores regress.

### Pitfall 5: Vet-intake markdown path traversal
**What goes wrong:** Attacker (or buggy simulator) provides `cow_tag="../../etc/passwd"`; drafter writes to `runtime/vet_intake/../../etc/passwd.md`.
**Why it happens:** Cow tags originate from world config (trusted) but the code path must still validate.
**How to avoid:** `re.fullmatch(r'[A-Z][0-9]{3}', cow_tag)` before concatenation; use `pathlib.Path(intake_dir, f"{cow_tag}_{iso_ts}.md").resolve()` and assert it starts with `intake_dir.resolve()`.
**Warning signs:** Any filename-assembly code that does `f"{user_input}.md"` without validation.

### Pitfall 6: Predator pulse ring's `threat-ring` keyframe defined but never applied
**What goes wrong:** `index.css:179-186` defines `.threat-ring` but `RanchMap.tsx:296-300` draws the ring directly in Canvas with static alpha. Polish PR "use threat-ring" and end up with two renderers fighting.
**Why it happens:** The CSS animation works on DOM elements; Canvas is a pixel buffer.
**How to avoid:** Pick ONE — either (a) RAF-interpolate `alpha` inside `RanchMap.draw()` based on elapsed ms (`alpha = (Math.sin(t/600) + 1) / 2`) keeping everything in Canvas, or (b) overlay absolutely-positioned divs with `.threat-ring` class outside the Canvas. Option (a) matches existing architecture; recommend it.
**Warning signs:** Two pulse timings competing or ring disappearing when canvas redraws.

### Pitfall 7: Cost-ticker "paused" state looks identical to "active" state on a 1080p video at low bitrate
**What goes wrong:** Judges watching compressed video miss the "PAUSED (idle)" chip + border color change because both are muted tones.
**Why it happens:** The sage accent is intentionally understated.
**How to avoid:** In addition to chip + border, **dim the cumulative cost number** (color `text-2` instead of `accent-sage`) and **freeze the sparkline** (draw last value as flat line in `text-2` color). Produces a visible "frozen dashboard" look when agents idle.
**Warning signs:** Watch the 3-min video at 480p on a phone — if you can't tell the meter is paused in 1 second, the treatment isn't strong enough.

   - What we know: `VISION.md` mentions a "Doc" concept; PROJECT.md lists `EXT-02` (Doc 6th agent) as README-roadmap-only deferred item.
   - What's unclear: For Phase 5, is the drafter a method on `HerdHealthWatcher` or a new skill file `skills/cattle-behavior/doc-vet-intake.md` that the LLM invokes?
   - Recommendation: **Inline drafter** in `HerdHealthWatcher` for this milestone (keeps "Doc as 6th agent" properly scoped to EXT-02). A skill file is overkill for a fixed-schema document.

2. **Does `draft_vet_intake` become an MCP tool (LLM-invokable) or a direct Python call from the handler?**
   - What we know: The sick-cow scenario assertion (`sick_cow.py:110-118`) already checks for `classify_pipeline` and `page_rancher` as tool names in `mesh` tool-call log.
   - What's unclear: If `draft_vet_intake` is NOT a tool, the assertion needs a different mechanism (e.g., check for file-on-disk). If it IS a tool, it becomes LLM-invokable and needs sim-path parity in `simulate.py`.
   - Recommendation: **Make it an MCP tool** in `rancher_mcp.py` — matches the "page_rancher" precedent, gives scenario tests a uniform assertion surface.

3. **Where does the `vet_intake.drafted` SSE event fire from?**
   - What we know: `EventBroadcaster` already has producer loops; `_broadcast` is the shared sink.
   - What's unclear: Do we poll `runtime/vet_intake/` (like the attest loop polls the ledger) or does the MCP tool publish directly to MQTT → broadcaster?
   - Recommendation: **Poll-based** to start — matches existing `_attest_loop` pattern. Can switch to push later without API change.

4. **What's the fallback if Lighthouse 90 is unattainable due to dev-tool/font size after polish?**
   - What we know: Current build is unmeasured; prior experiments with similar stacks land 85-95 depending on optimizations.
   - What's unclear: Whether we can hit 90 without sacrificing motion/polish.
   - Recommendation: Measure **before** polish work, re-measure after, and if 90 isn't met, escalate to Claude's discretion which wins to take (font pair trim, remove `@fontsource` imports that aren't used above-fold, etc.).

5. **Do we expose `/api/attest/verify` without auth?**
   - What we know: Current dashboard has no auth (CORS-restricted to localhost).
   - What's unclear: Verify is O(N) over the whole ledger — could be a DoS vector if exposed publicly. For the hackathon demo, CORS localhost restriction is enough.
   - Recommendation: Leave unauth'd for this milestone; note in security comments. Year-2 LRP-rider hardening (`ATT-01`) is where this gets auth.

### Known Threat Patterns for {FastAPI + React SSE + SQLite ledger}

| Pattern | STRIDE | Standard Mitigation |
|---------|--------|---------------------|
| Path traversal on `cow_tag` → vet-intake filename | Tampering | Regex validation + `pathlib.Path.resolve()` containment assertion (Pitfall 5) |
| SSE connection exhaustion / DoS | Denial of Service | `_SSE_MAX_CONNECTIONS` semaphore already caps at 100 (existing in `app.py:54-56`) |
| Slow subscriber blocks broadcaster | Denial of Service | Bounded `asyncio.Queue(maxsize=100)` per subscriber, drop-oldest on full (existing) |
| `/api/attest/verify` O(N) scan DoS | Denial of Service | CORS localhost-only + demo-scale ledgers (<10k events); revisit in year-2 ATT-01 |
| Markdown injection via `signals` list (e.g., `"][malicious]"`) | XSS (if rendered in browser) | Escape-render markdown; avoid `dangerouslySetInnerHTML`; use DOMPurify or a known-safe renderer. For 20-line inline renderer, strictly whitelist tags. |
| Private-key exfiltration via `/api/attest/verify` response | Information Disclosure | `VerifyResult` schema excludes private key — already correct; `Signer.__repr__` elides key material (existing) |
| SQL injection on `since_seq` | Tampering | Already parameterized query in `ledger.py:220-229` — existing safe |
| SSE-response header buffering breaks real-time delivery | Integrity (of timing) | Add `X-Accel-Buffering: no` to `EventSourceResponse` headers (Pitfall 1) |
