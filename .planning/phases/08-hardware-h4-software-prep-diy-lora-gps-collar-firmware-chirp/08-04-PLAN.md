# Plan 08-04 — H4 Runbook + Full Bridge Integration Test + Determinism

## Objective

Close Phase 8 with:

1. `docs/HARDWARE_H4_RUNBOOK.md` — numbered-step runbook from blank RAK3172 to dashboard pin.

3. Full-suite coverage + determinism check (3× `make demo SEED=42`).

## Tasks

### Task 1 — `docs/HARDWARE_H4_RUNBOOK.md` (type="auto")

Create the runbook with these sections:

1. **Overview** — one paragraph. What this runbook does, time estimate, skills required.
2. **Prerequisites** — checklist:
   - ChirpStack v4 instance reachable (docker-compose one-liner included)
   - Mosquitto broker reachable
   - PlatformIO installed (`pip install platformio`)
   - Python 3.11+ + `uv sync` already run
3. **Hardware BOM** — link to `hardware/collar/BOM.md`
4. **Step 1 — Assemble the collar** — link to `hardware/collar/wiring.md` + `README.md`
5. **Step 2 — Flash the firmware** — use `./hardware/collar/flash.sh`
6. **Step 3 — Launch ChirpStack** — include `docker-compose.yml` snippet (official `chirpstack/chirpstack-docker` image, one-liner command). Link out to upstream `https://github.com/chirpstack/chirpstack-docker` for deep config.
7. **Step 4 — Create application + device profile in ChirpStack** — screenshots paths (not included — just referenced as TBD placeholders). DevEUI + AppKey copied to `hardware/collar/firmware/include/secrets.h`.
8. **Step 5 — Register the collar in SkyHerd** — run `python hardware/collar/provisioning/register-collar.py --dev-eui ... --ranch ranch_a --cow-tag A001`.
9. **Step 6 — Start the ChirpStack bridge** — `python -m skyherd.edge.chirpstack_bridge --ranch ranch_a --mqtt-host localhost` (also document `systemd` unit snippet for persistent run).
10. **Step 7 — Verify on dashboard** — expected behavior: pin turns green within one uplink interval (~15 min default, `--interval 60` during bring-up).
11. **Troubleshooting** — top 5 failure modes:
    - No OTAA join → check AppEUI / AppKey mismatch, LoRaWAN region, antenna
    - Uplink visible in ChirpStack but not in bridge → check MQTT integration enabled + app_id
    - Bridge publishes but dashboard doesn't update → check cow_tag registered in `runtime/collars/registry.json`
    - Battery drains fast → confirm GPS_PWR_PIN wired to MOSFET gate + BATSAVE active
    - Collar disappears from dashboard → stale fix_age > 3600 s filter; power-cycle the collar.
12. **Appendix A — ChirpStack docker-compose snippet** — self-contained.
13. **Appendix B — Known deferred items** — LoRaWAN FUOTA OTA, BLE DFU, multi-region antenna stocking, geofence-driven uplink frequency.

Keep under 500 lines.

### Task 2 — `tests/hardware/test_h4_end_to_end.py` (type="auto")

Integration-style test (still in-process, no real MQTT). Chain:

`CollarSimEmitter.tick()` → get dict → encode via `encode()` → base64 wrap → build ChirpStack uplink JSON → `ChirpStackBridge.handle_raw_event` → assert MQTT publish topic + schema matches what `CollarSensor` would emit.

Tests:
- `test_sim_to_bridge_round_trip_schema_match` — single tick end-to-end.
- `test_sim_to_bridge_multi_cow_isolation` — 3 emitters (different seeds, different cow_tags) → 3 distinct topics, no cross-pollination.
- `test_sim_to_bridge_determinism` — same seed produces byte-identical MQTT payloads across two runs (modulo `ts` field).
- `test_sim_to_bridge_sim_and_real_schemas_agree` — `CollarSensor` sim payload keys == bridge-decoded payload keys (set equality on known subset).
