# Plan 08-02 ‚Äî ChirpStack Bridge (uplink ‚Üí MQTT republisher)

## Objective

Build `src/skyherd/edge/chirpstack_bridge.py` ‚Äî an async bridge that subscribes to ChirpStack v4 uplink MQTT events, decodes the 16-byte LoRaWAN payload, and republishes on the canonical `skyherd/{ranch}/collar/{cow_tag}` topic. Zero new runtime deps. Injectable client for tests.

## Tasks

### Task 1 ‚Äî Create `chirpstack_bridge.py` (type="auto")

Create `src/skyherd/edge/chirpstack_bridge.py` with:

- Module docstring referencing ChirpStack v4 MQTT integration topic pattern: `application/{app_id}/device/{dev_eui}/event/up`.
- `ChirpStackUplink` dataclass (frozen) ‚Äî `dev_eui: str`, `app_id: str`, `f_port: int`, `f_cnt: int`, `payload_b64: str`, `rx_ts: float`.
- `_parse_uplink_event(raw: bytes | dict) -> ChirpStackUplink | None` ‚Äî accepts the ChirpStack JSON event, extracts relevant fields, returns None on malformed.
- `CollarRegistry` helper ‚Äî loads `runtime/collars/registry.json` (path env-configurable via `SKYHERD_COLLAR_REGISTRY`). Maps `dev_eui` ‚Üí `(ranch_id, cow_tag)`. Graceful fallback when registry missing (logs + returns None).
- `ChirpStackBridge` class:
  - `__init__(self, *, mqtt_publish: Callable[[str, bytes], Awaitable[None]], registry: CollarRegistry, decoder: Callable[[bytes, str, str], CollarReading], ts_provider: Callable[[], float] | None = None)`
  - `async handle_raw_event(raw: bytes | dict) -> bool` ‚Äî parses, decodes, resolves cow, publishes; returns True on success, False on skipped/malformed (logged).
- `async run_forever(client)` helper ‚Äî subscribes a client (any object with `async uplinks() -> AsyncIterator[bytes]`) and pumps events through `handle_raw_event`.

Use `decode_payload.decode` via dynamic import (since provisioning is not a package). Extract the import helper so tests can monkeypatch.

### Task 2 ‚Äî Real ChirpStack MQTT client (thin wrapper) (type="auto")

In the same module, add `ChirpStackMqttClient`:
- `__init__(host, port, username, password, app_id, *, tls=False)`
- `async __aenter__` / `__aexit__` manages an `aiomqtt.Client` context.
- `async uplinks() -> AsyncIterator[bytes]` ‚Äî subscribes to `application/{app_id}/device/+/event/up`, yields raw message payloads.

This class is stubbed out in tests; unit-test coverage focuses on `ChirpStackBridge` + `_parse_uplink_event` + `CollarRegistry`.
