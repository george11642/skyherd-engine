# Plan 08-01 — Firmware Polish (battery save, GPS duty cycle, OTA placeholder)

## Objective

Polish `hardware/collar/firmware/src/main.cpp` for field deployment: add GPS power-gating, battery-save interval extension, OTA update placeholder, and a structured docs block. Keep firmware < 500 lines and buildable via `pio run -e native -t checkprogsize` (or defer to manual verify).

## Tasks

### Task 1 — Top-of-file docs block upgrade (type="auto")

Add a structured docs block above the existing comment describing:
- Pin map (GPS UART, IMU I2C, battery ADC, GPS VCC control, LED)
- Power tree (LiPo → TP4056 → 3V3 LDO → RAK3172 + GPS + IMU)
- Power budget (active draw ~120 mA for 90 s GPS fix + 10 s IMU + 2 s LoRa uplink; sleep draw ~10 µA)
- LoRaWAN region (US915 default; EU868 via `platformio.ini` flag)
- Battery-save behavior (extends interval 4× below 15%)

Keep concise — single comment block, ~40 lines max.

**Verify:** `head -60 hardware/collar/firmware/src/main.cpp` shows the new docs block.
**Done:** docs block compiles (no syntax errors); total file remains < 500 lines.

### Task 2 — GPS power-gating (type="auto")

Add a `#define GPS_PWR_PIN PA2` (overrideable via build flag). In `setup()`, configure `pinMode(GPS_PWR_PIN, OUTPUT)` + `digitalWrite(GPS_PWR_PIN, HIGH)` to power on GPS. Create `gps_power_off()` / `gps_power_on()` helpers. Call `gps_power_off()` right before `api.system.sleep.all(...)` and `gps_power_on()` + serial reinit at the top of each loop iteration.

If `GPS_PWR_PIN` is not defined (e.g. on boards without the MOSFET), the helpers compile as no-ops via `#ifdef GPS_PWR_PIN` gate.

**Verify:** grep for `gps_power_off` and `gps_power_on` — each called exactly once per cycle. No pin control on unsupported boards.
**Done:** compile path is guard-clean; helpers use `#ifdef GPS_PWR_PIN` for portability.

### Task 3 — Battery-save interval extension (type="auto")

In `loop()`, after computing `battery_pct`, if `battery_pct < LOW_BATTERY_THRESHOLD_PCT`, use `send_interval_ms * BATSAVE_MULTIPLIER` (default 4) for the sleep call. Log `[batsave] extending interval 4x to %lu ms`. Add `#define BATSAVE_MULTIPLIER 4` with build-flag override.

**Verify:** grep confirms `BATSAVE_MULTIPLIER` is used in the `api.system.sleep.all(...)` branch and surfaced in `platformio.ini`.
**Done:** firmware handles low-battery gracefully; tests (if any native test exists) still pass.

### Task 4 — OTA update placeholder (type="auto")

Add a structured `#ifdef OTA_ENABLED` block near the end with:
- TODO: LoRaWAN FUOTA (Fragmentation + Multicast) — link to RAK AT command reference
- TODO: BLE DFU via RAK RUI3 bootloader — reference RAK DFU example repo
- TODO: file-transfer protocol choice (FUOTA McGroup vs multicast)

Not implemented — just documented sign-posts. Remove the existing single-line `OTA_ENABLED` stub.

**Verify:** grep for `TODO` inside `#ifdef OTA_ENABLED` block returns ≥ 3 lines; no actual BLE calls.
**Done:** block is commented, compiles (when `OTA_ENABLED` undefined).

### Task 5 — platformio.ini build flags update (type="auto")

Add to `[env:rak3172]` `build_flags`:
- `-D GPS_PWR_PIN=PA2`
- `-D BATSAVE_MULTIPLIER=4`

Add a comment block above `build_flags` describing each flag.

For `[env:heltec]`: no `GPS_PWR_PIN` (ESP32 has power pin differently) — explicit comment.

**Verify:** `cat hardware/collar/firmware/platformio.ini` shows both flags in rak3172 env.
**Done:** file valid INI syntax.

### Task 6 — Smoke-check native test compilation (type="auto")

Check if `pio` is available: `which pio`. If yes, run `cd hardware/collar/firmware && pio test -e native 2>&1 | tail -20`. If no, check for `arduino-cli` — if neither available, document in a comment in the test file: "Requires PlatformIO native env; skipped in sandbox." Do NOT install platformio in CI.

Also verify `hardware/collar/firmware/test/test_main.cpp` exists and is syntactically valid C++.

**Verify:** If pio available — test passes. Otherwise, test file exists + documents skip reason.
**Done:** no broken native test harness.
