# Plan 08-03 — Flash Script, BOM Finalization, Collar Simulator

## Objective

Three deliverables:

1. `hardware/collar/flash.sh` — one-shot flash wrapper with safety checks.
2. `hardware/collar/BOM.md` — polished with exact supplier URLs, shipping notes, regulatory note, and "can I skip" column.
3. `src/skyherd/sensors/collar_sim.py` — deterministic seed-driven collar emitter, distinct from World-attached `CollarSensor`.

## Tasks

### Task 1 — `hardware/collar/flash.sh` (type="auto")

Create an executable bash script at `hardware/collar/flash.sh`:
- Shebang: `#!/usr/bin/env bash`
- `set -euo pipefail`
- Accept flags: `--env rak3172|heltec` (default rak3172), `--monitor` (tail serial after flash), `--no-warn` (skip boot-mode prompt).
- Pre-flight checks:
  - Verify `pio` binary on PATH. If missing, print install instructions and exit 2.
  - Verify `hardware/collar/firmware/include/secrets.h` exists. If not, copy `secrets.h.example` to `secrets.h` and warn the user to fill in creds.
- Prompt user (unless `--no-warn`): "Hold BOOT0, press RESET, release BOOT0. [Enter to continue]".
- Run `pio run -e $ENV -t upload` from the firmware dir.
- If `--monitor`, follow with `pio device monitor --baud 115200` for 20s then exit.
- Clear error messages with exit codes: 0 success, 2 missing tools, 3 flash failure.
- Script under 120 lines.

Mark executable: `chmod +x hardware/collar/flash.sh`.

### Task 4 — `src/skyherd/sensors/collar_sim.py` (type="auto")

Create `src/skyherd/sensors/collar_sim.py`:

A standalone deterministic collar emitter — useful for dashboard dev and for augmenting partial hardware deployments.

- `class CollarSimEmitter`:
  - `__init__(self, *, ranch_id, cow_tag, seed, start_pos=(34.05, -106.53), drain_rate_per_tick=0.05, ts_provider=None)`
  - Uses `random.Random(seed)` — no global state, no wall clock.
  - `tick() -> dict` — returns one synthetic `collar.reading` payload matching sim schema:
    - `ts`, `kind="collar.reading"`, `ranch`, `entity`, `pos`, `heading_deg`, `activity`, `battery_pct`, optional `heart_rate_bpm`.
  - Position: random walk — each tick `pos += rng.gauss(0, 2e-5)` in lat/lon.
  - Activity state machine: three states (resting, grazing, walking) with seed-driven transition probabilities (resting→grazing 0.3, grazing→walking 0.2, walking→grazing 0.4, else stay). Initial state = resting.
  - Heart rate: Ornstein-Uhlenbeck pullback around 55 bpm with `rng.gauss(0, 3)`.
  - Battery: drains `drain_rate_per_tick` per tick, clamped [0, 100].
  - `ts_provider` defaults to `lambda: float(self._tick_count)` — deterministic monotonic int.

- `async def run_async(emitter, bus, count, *, tick_interval_s=0.0)` — top-level coroutine that ticks N times and publishes to `skyherd/{ranch}/collar/{cow_tag}` via the injected bus.

Keep under 200 lines. Type-annotated, ruff-clean.

### Task 5 — Collar sim tests (type="auto")

- `test_emitter_activity_valid` — activity always in `{resting, grazing, walking}`.
- `test_emitter_position_drifts_bounded` — 1000 ticks; position delta < 1 km from start.
- `test_emitter_determinism_same_seed` — seed 42, 100 ticks → list of dicts is byte-identical across two instances.
- `test_emitter_determinism_different_seed` — seed 42 vs seed 43 → different output.
- `test_run_async_publishes_n_messages` — `run_async(emitter, mock_bus, count=5)` → 5 messages on expected topic.

Use a lightweight mock bus (existing `mock_bus` fixture from `tests/sensors/conftest.py` if reusable).
