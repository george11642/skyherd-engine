# Phase 7.2 — iOS Swift hardening + iOS↔Android parity (audit-fix pass)

**Extension of:** Phase 7 (H3 — Mavic Air 2 hardening) and Phase 7.1 (laptop path).
**Target:** close every code-reality gap surfaced by three parallel audits of the
iOS companion app, Android companion app, and iOS build system. Audit-driven,
not plan-driven — this phase was defined retroactively by the fix-agent brief.
**Deadline:** 2026-04-26 20:00 EST.

---

## Why decimal (7.2 not 11)

Phase 7 shipped the functional MavicAdapter + MAVSDK failover backend and
Phase 7.1 shipped the laptop-primary HTTP surface. Parallel audits ran the
same afternoon against the companion-app code to make sure the *secondary*
phone path was actually correct — not just dead-weight documentation. The
audits found 28 fix items across 3 categories:

1. **iOS code reality** (12 items) — BatteryGuard off-by-one, silent RTH
   masquerading as `gotoLocation`, MQTTBridge never started at launch, no
   lost-signal watchdog, no MissionV1 envelope decoder, unbounded seq window,
   dead WebSocket refs, dev-Arm button that did nothing, UIDevice shim, etc.
2. **iOS↔Android parity** (6 items) — Android battery floor 25 % vs iOS 30 %,
   wind operator `>` vs `>=`, geofence bounding-box vs ray-cast, topic
   scheme drift (`skyherd/drone/state/<id>` vs `skyherd/drone/state/android`),
   missing metadata consumption from MissionV1 envelope.
3. **iOS build system** (10 items) — `DJI_SDK_AVAILABLE` flag never wired,
   missing `Config.xcconfig.template`, unused networkextension entitlement
   blocking personal-team sideloading, Simulator-only CI build that masked
   device-slice failures, missing IPA packaging for Sideloadly.

A decimal phase keeps the change localized to the 7-series hardware surface
and avoids reshuffling the roadmap two days before submission.

## Scope (five parts, one commit each)

### Part A — iOS Swift hardening (commit 8fe069d)
12 critical iOS code fixes. Adds `LostSignalWatchdog.swift`, rewrites
`CommandRouter` envelope parsing + seq eviction, wires MQTT + watchdog at
`App` launch, tightens `DJIBridge` to throw on unimplemented operations
instead of silently calling `startGoHome`.

### Part B — Android parity (commit 2e3cd22)
6 parity fixes to `SafetyGuards.kt`, `DroneControl.kt`, `MQTTBridge.kt`.
Battery reject flips to `<=`, wind to `>=`, geofence from bbox to ray-cast,
MissionV1 envelope parser consumes `metadata.battery_floor_pct`/`wind_kt`/
`geofence_version`, topics unified to `skyherd/drone/{ack,state}/android`.

### Part C — iOS build system (commit 412086c)
10 build-system fixes. `project.yml` wires `SWIFT_ACTIVE_COMPILATION_CONDITIONS`
from xcconfig so `DJI_SDK_AVAILABLE=1` actually lights up the real SDK path.
New `Config.xcconfig.template`. networkextension entitlement removed.
`ios-app.yml` builds device slice (not Simulator-only) + packages unsigned
IPA for Sideloadly.

### Part D — Docs alignment (commit 695e60a)
4 runbooks updated to match code reality: README drops WebSocket refs + adds
Sideloadly no-Mac path + DJI SDK 5.11.x pin; HARDWARE_MAVIC_IOS gets two-path
table; HARDWARE_H3_RUNBOOK §9 Path C replaces `.app/simctl` claim with IPA +
Sideloadly path; MAVIC_MISSION_SCHEMA §5 adds unified MQTT topic scheme +
lost-signal watchdog contract.

### Part E — Parity unit tests (commit 89bce50)
Net-new unit tests: iOS +18 (CommandRouterTests, LostSignalWatchdogTests,
DJIBridgeTests, MQTTBridgeTests), Android +18 (SafetyGuardsTest,
DroneControlEnvelopeTest). Adds `junit` + `kotlin-test` +
`kotlinx-coroutines-test` to Android `build.gradle.kts`.

## Hard constraints

- **Sim-first still sacred:** no Python code touched. The 1731-test Python
  suite must remain green. Verified by not modifying any `src/skyherd/**/*.py`.
- **No real-flight tests:** all iOS/Android tests run in stub mode (no DJI
  SDK artifact on CI runners). Real hardware validation is the Friday field
  test on the laptop path — 7.2 code is a parallel safety-guard layer.
- **MIT + zero attribution** — global git config enforces.
- **Five commits, one per part** — keeps the diff reviewable.
