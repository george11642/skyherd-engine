### Task 1 — New `docs/LAPTOP_DRONE_CONTROL.md` (`type="auto"`)

Sections:
1. **Why laptop-as-controller** — no Mac, no phone, single USB-C cable, same MAVSDK path used in CI tests.
2. **Cable spec** — USB-C to USB-C, data + power (not charge-only). Link to a known-good Anker SKU. Note: USB-A adapter works but adds one failure point.
3. **Which port on the Mavic** — **USB-C on the RC controller** (not the aircraft). Diagram described in prose with a tiny ASCII callout.
4. **Linux (Ubuntu 24.04 primary)** — udev rules at `/etc/udev/rules.d/99-skyherd-drone.rules`, reload with `sudo udevadm control --reload && sudo udevadm trigger`, verify `lsusb | grep -i dji` then `ls -l /dev/ttyACM*`.
5. **WSL2 (secondary)** — `usbipd-win` on the Windows host, `usbipd list` → `usbipd bind --busid X-Y` → `usbipd attach --wsl --busid X-Y`. Call out: **not validated in the sandbox; manual verify Friday morning.**
6. **Windows native** (skipped — user runs Ubuntu per `/etc/os-release`. Note: "see WSL2 section above if running on Windows 11".)
7. **MAVSDK install** — `pip install mavsdk pymavlink` (already in `pyproject.toml` extras). If missing binary: `mavsdk` wheel ships its own `mavsdk_server`, no apt-install needed.
8. **Troubleshooting** — no device detected / permission denied / heartbeat timeout (each with `verify:` + `fix:`).
9. **Test it:** `make laptop-drone-smoke` — runs the mocked end-to-end chain in < 10 s.
10. **Friday morning workflow** — power on RC → plug USB-C → run `DRONE_BACKEND=mavic_direct uv run python -m skyherd.server.live --port 8000` → open dashboard → enable `?drone=1` → ARM + TAKEOFF.

Include a copy-pasteable udev rules block:

```
# /etc/udev/rules.d/99-skyherd-drone.rules
# DJI Mavic / Parrot / generic MAVLink USB-serial devices
SUBSYSTEM=="tty", ATTRS{idVendor}=="2ca3", MODE="0666", GROUP="plugdev"
SUBSYSTEM=="tty", ATTRS{idVendor}=="26ac", MODE="0666", GROUP="plugdev"
SUBSYSTEM=="tty", ATTRS{idVendor}=="0483", MODE="0666", GROUP="plugdev"
```

Add user to `plugdev` group: `sudo usermod -aG plugdev $USER && newgrp plugdev`.

Verify: `test -f docs/LAPTOP_DRONE_CONTROL.md && wc -l docs/LAPTOP_DRONE_CONTROL.md` → > 150 lines.
