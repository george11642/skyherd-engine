# Plan 07-03 — Versioned mission JSON schema (Pydantic v1)

## Objective

Ship `src/skyherd/drone/mission_schema.py` (Pydantic models) +
`docs/MAVIC_MISSION_SCHEMA.md` (spec) for the JSON that the agent emits
and the companion apps consume. Wire `MavicAdapter.patrol` to serialize
incoming `list[Waypoint]` into a `MissionV1` before dispatching to the
active leg. Forward-compat via `extra="allow"` on top-level. Coverage
≥ 85 %.

## Context

- `src/skyherd/drone/interface.py` already defines a `Waypoint` Pydantic
  model — reuse it, don't duplicate.
- `docs/HARDWARE_MAVIC_PROTOCOL.md` describes the wire format informally
  today. The new schema doc supersedes the "mission payload" section of
  that doc (we'll add a "See MAVIC_MISSION_SCHEMA.md" pointer).
- Companion apps currently parse per-field (`wp["lat"]`, `wp["lon"]`,
  `wp["alt_m"]`, `wp["hold_s"]`). The schema is a superset — adding
  `mission_id`, `version`, `metadata` (wind, battery floor, geofence
  version), and `failover` hints — so existing apps continue working.
  The schema's top-level `extra="allow"` field ensures we can add
  fields in future versions without breaking older apps.

## Implementation

### File 1 — `src/skyherd/drone/mission_schema.py` (new, ~180 lines)

```python
"""Versioned mission JSON schema for agent → companion-app handoff.

The agent produces ``MissionV1`` objects; the MavicAdapter serialises
them to JSON and forwards to either Leg A (DJI via companion app) or
Leg B (MAVSDK via USB-C OTG). Both legs parse the same schema.

Versioning policy
-----------------
* ``version`` is a literal int.  Bumping means a breaking change to the
  payload; old companion apps will reject the mission.
* Non-breaking additions (new optional fields) do NOT bump ``version``
  thanks to ``extra="allow"`` on the top-level model.
* Removing or renaming a field IS breaking — bump ``version``.

Current version: 1.
"""

from __future__ import annotations

from typing import Any, Literal

from pydantic import BaseModel, ConfigDict, Field

from skyherd.drone.interface import Waypoint

__all__ = [
    "FailoverHint",
    "MissionMetadata",
    "MissionV1",
    "SCHEMA_VERSION",
]

SCHEMA_VERSION: Literal[1] = 1


class FailoverHint(BaseModel):
    """Optional hint telling the companion app which leg to prefer."""
    preferred_leg: Literal["dji", "mavsdk", "auto"] = "auto"
    retry_on_other_leg: bool = True
    retry_budget: int = 1


class MissionMetadata(BaseModel):
    """Per-mission safety + attestation metadata."""
    mission_id: str = Field(..., min_length=1, max_length=64)
    ranch_id: str = Field(..., min_length=1, max_length=64)
    scenario: str | None = None            # e.g. "coyote_fence", "wildfire"
    wind_kt: float | None = None
    battery_floor_pct: float = 30.0
    geofence_version: str | None = None    # matches ranch world YAML
    issued_ts: float | None = None         # epoch seconds, monotonic in replay
    issued_by: str | None = None           # agent id


class MissionV1(BaseModel):
    """Top-level mission packet.

    ``extra="allow"`` keeps forward compatibility — newer fields added
    in future minor revisions remain parseable by older apps (they'll
    simply ignore unknown fields).
    """
    model_config = ConfigDict(extra="allow")

    version: Literal[1] = SCHEMA_VERSION
    metadata: MissionMetadata
    waypoints: list[Waypoint] = Field(..., min_length=1, max_length=64)
    deterrent_tone_hz: int | None = Field(None, ge=500, le=20000)
    deterrent_duration_s: float | None = Field(None, ge=0.0, le=60.0)
    failover: FailoverHint = Field(default_factory=FailoverHint)

    def to_wire(self) -> dict[str, Any]:
        """JSON-compatible dict for MQTT / WebSocket dispatch."""
        return self.model_dump(mode="json")

    @classmethod
    def from_wire(cls, payload: dict[str, Any]) -> "MissionV1":
        return cls.model_validate(payload)
```

### File 3 — `src/skyherd/drone/mavic_adapter.py` (edit)

Add `patrol_mission(mission: MissionV1)` method (parallel to existing
`patrol(waypoints)`). Existing `patrol(waypoints)` internally builds a
default `MissionV1` with synthetic metadata so the schema flows through
on every call — this becomes the canonical path.

```python
async def patrol(self, waypoints: list[Waypoint]) -> None:
    mission = MissionV1(
        metadata=MissionMetadata(
            mission_id=f"mission_{self._next_seq():08x}",
            ranch_id=self._ranch_id,
        ),
        waypoints=list(waypoints),
    )
    return await self.patrol_mission(mission)

async def patrol_mission(self, mission: MissionV1) -> None:
    # Existing failover logic on waypoints list
    return await self._invoke("patrol", mission.waypoints)
```

Ledger entries now include `mission_id` field.
