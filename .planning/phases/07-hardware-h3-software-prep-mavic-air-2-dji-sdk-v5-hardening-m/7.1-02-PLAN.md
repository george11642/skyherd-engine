# 7.1-02 ‚Äî LaptopDroneControl panel + hold-to-arm UX

**Objective:** Dashboard panel that mounts the backend endpoints from 7.1-01. Hidden-by-default (no surprise clicks), hold-to-arm on dangerous actions, sage/dust styling, full vitest coverage.

## Context references

- `App.tsx` already uses `RightRailAccordion` with 4 tabs (Memory / Attestation / Vet / Cross-Ranch).
- Styling tokens: `var(--color-bg-1)`, `var(--color-line)`, `chip chip-sage | chip-danger | chip-muted`.
- SSE via `getSSE()` ‚Äî `drone.manual_override` event will update state chips.

## Tasks

### Task 1 ‚Äî `web/src/components/LaptopDroneControl.tsx` (`type="auto"`)

Build a self-contained panel:

- Props: `{ collapsed?: boolean; onToggle?: () => void; token?: string }`. Token defaults to reading `window.__DRONE_MANUAL_TOKEN` (string) ‚Äî tests inject via global stub.
- **Top band:** state chips row ‚Äî `Battery %`, `GPS valid/invalid`, `Signal strength`, `Mode` (agent / manual / grounded). Polls `/api/snapshot` every 1000 ms, reads `data.drone.{battery_pct, lat, lon, state}`. Derives `GPS valid` = `lat!=0 && lon!=0`. Mode chip colored: sage when "agent", dust when "manual", muted when "grounded".
- **Button grid (2 cols √ó 3 rows):** ARM, DISARM, TAKEOFF, RTL, LAND, EMERGENCY STOP. Dangerous actions (ARM, TAKEOFF, ESTOP) require hold-to-fire: `pointerdown` starts a 3 s progress timer, `pointerup/leave` before 3 s aborts. Progress rendered as a thin bar inside the button.
- **Hold-to-arm accessibility:** for keyboard users, hold `Space` or `Enter` for 3 s with focus on the button (use `keydown`/`keyup` listeners + same timer). Emit ARIA live-region announcement when fire completes: `"{action} fired"`.
- **Action handler:** on fire, POST to `/api/drone/{action}` with header `X-Manual-Override-Token: {token}`, empty JSON body. Show inline status: in-flight ‚Üí "‚Ä¶sending" (chip-muted), 2xx ‚Üí "sent" (chip-sage), 4xx/5xx ‚Üí "error: {status}" (chip-danger).
- **SSE subscription:** listen to `drone.manual_override`; on matching action echo, update an action history strip below the grid (last 5 actions with timestamp + actor + latency).
- **Visibility gating:** panel returns `null` unless `window.__DRONE_MANUAL_ENABLED === true` OR the URL has `?drone=1` on mount. Use a tiny helper `isManualEnabled(): boolean`.
- Styling mirrors `MemoryPanel.tsx` ‚Äî outer `<section>` with `rounded border`, header with `button` that calls `onToggle`, aria-expanded / aria-controls.

Verify: `(cd web && pnpm run build)` ‚Üí bundle compiles. No new deps.
Done when: component renders in jsdom with all 6 buttons + state chips.
