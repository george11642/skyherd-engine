# 7.1-01 — Backend manual-override API + USB-C smoke

**Objective:** Stand up the HTTP surface the dashboard panel (plan 02) will call. Endpoints are token-gated, idempotent, and emit SSE events. Verify MAVSDK-over-USB-C wiring end-to-end via mocks.

## Context references

- Right-rail panels expect to talk to `/api/drone/*` endpoints.
- `PymavlinkBackend` already speaks ARM / TAKEOFF / RTL at the wire level.
- SSE broadcaster exposes `_broadcast(event_type, payload)` and a public async wrapper pattern (`emit_memory_written`).

## Tasks

### Task 1 — Create `src/skyherd/server/drone_control.py` (`type="auto"`)

Build a new module that factors the drone-control surface out of `app.py`:

- `attach_drone_control(app, *, get_backend, broadcaster, token)` — mounts the five POST routes onto the FastAPI app.
- `ManualAction` enum: `arm | disarm | takeoff | rtl | land | estop`.
- `ManualOverrideRequest(BaseModel)` with optional `alt_m: float` (0 < alt ≤ 120).
- Token check via FastAPI dependency reads `X-Manual-Override-Token`; compares to `token` arg. Missing → 401, mismatch → 403, empty configured token → 503 ("manual override disabled").
- Backend check: if `get_backend()` returns None → 503 "no drone attached".
- Each endpoint:
  1. Check token (via dependency)
  2. Resolve backend (or 503)
  3. Invoke the backend method (`takeoff`, `return_to_home`, `disarm`, etc.) — wrap in try/except mapping `DroneUnavailable` → 503, `DroneError` → 502.
  4. Record wall start/end, broadcast `drone.manual_override` payload with `action, actor="laptop", ts, success, latency_ms, error?`.
  5. Return `{ok: true, action: ..., latency_ms: ...}`.
- `ARM` and `DISARM` must be a no-op when the state already matches (idempotent). `ESTOP` triggers `return_to_home()` first; if that fails → `disarm()`; if that fails → still broadcast + return 200 with `best_effort: true`.
- `LAND` calls `return_to_home()` (Mavic's land = RTL from current hover point on our backends).
- No mutable module-level state — everything closes over locals in `attach_drone_control`.

Verify: import the module and confirm symbols exist:
```bash
uv run python -c "from skyherd.server.drone_control import attach_drone_control, ManualAction; print('ok')"
```
Done when: import succeeds, no pyright complaints.

### Task 2 — Wire into `create_app()` (`type="auto"`)

In `src/skyherd/server/app.py`:

- Read `SKYHERD_MANUAL_OVERRIDE_TOKEN` at app build time (default empty).
- Accept a `drone_backend` kwarg on `create_app()` (default None).
- After the memory API mount, call `attach_drone_control(app, get_backend=lambda: drone_backend, broadcaster=broadcaster, token=token)`.
- Add a minimal `get_manual_override_token()` helper for tests.

Verify: `uv run python -c "from skyherd.server.app import create_app; app = create_app(mock=True); print([r.path for r in app.routes if '/drone' in r.path])"` prints the 5 routes plus any existing.
Done when: routes mounted, mock mode continues to work (no drone backend → 503 on call, but /api/snapshot untouched).

### Task 3 — New `tests/server/test_drone_control.py` (`type="auto"`)

Eight tests using `httpx.AsyncClient` + `pytest_asyncio`:

1. `test_arm_without_token_returns_401`
2. `test_arm_with_wrong_token_returns_403`
3. `test_arm_without_backend_returns_503`
4. `test_arm_with_backend_calls_backend_and_broadcasts` (fake backend + fake broadcaster)
5. `test_double_arm_is_idempotent`
6. `test_estop_calls_rtl_then_disarm_on_failure`
7. `test_takeoff_rejects_out_of_range_alt`
8. `test_manual_override_token_disabled_returns_503`

Use a `FakeDroneBackend` subclass of `DroneBackend` with awaitable stub methods that record calls. Inject a `FakeBroadcaster` with a `.events: list[tuple[str, dict]]`.

Verify: `uv run pytest tests/server/test_drone_control.py -v` → 8 passing.
Done when: all 8 pass, coverage ≥85% on `drone_control.py`.

### Task 4 — New `tests/hardware/test_laptop_drone_control.py` (`type="auto"`)

One end-to-end test that validates the `DRONE_BACKEND=mavic_direct` path via mocks:

- `test_mavsdk_over_usb_c_smoke_mocked`: patches `pymavlink.mavutil.mavlink_connection` with a `MagicMock`. Calls `get_backend("mavic_direct")` → `PymavlinkBackend`, awaits `.connect()`, then `.takeoff(alt_m=20)`. Asserts `command_long_send` was called with `MAV_CMD_COMPONENT_ARM_DISARM` then `MAV_CMD_NAV_TAKEOFF`.
- `test_estop_chain_rtl_then_disarm`: exercises the ESTOP fallback through the HTTP endpoint with a FakeBackend that raises `DroneError` on RTL — asserts `disarm` was invoked as fallback.

Verify: `uv run pytest tests/hardware/test_laptop_drone_control.py -v` → 2 passing in < 2 s.
