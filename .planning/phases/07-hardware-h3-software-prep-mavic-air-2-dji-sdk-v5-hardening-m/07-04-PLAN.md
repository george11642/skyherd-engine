### File 2 — `tests/hardware/test_h3_dji_replay.py` (new, ~250 lines)

Fake `_WSTransport` wrapper that iterates through the fixture on each
`send_command`. Mocks the MAVSDK leg (`PymavlinkBackend`) with a
plain `AsyncMock` set to return `DroneState` on `state()` and `None`
on actuators.

Tests:
1. `test_replay_happy_path_no_failover` — truncated fixture ending
   before the error line → mission completes on DJI leg; no failover.
2. `test_replay_failover_on_signal_lost` — full fixture; adapter fails
   over to MAVSDK on line 18; completes mission; failover counter = 1.
3. `test_replay_ledger_chain_integrity` — collect every ledger entry
   during the replay; assert ordered kinds:
   ``[leg_selected, takeoff, patrol, failover, patrol, rth, disconnected]``
   (adapter- and pi-level entries merged in monotonic order).
4. `test_replay_wall_time_under_500ms` — timed run; fixture replay
   completes in < 0.5 s.
5. `test_replay_mission_id_matches_fixture` — every patrol entry in
   the ledger has the same `mission_id` even after failover.
6. `test_replay_deterministic_three_runs` — run the replay thrice with
   fixed seed; ledger entry list is byte-identical after
   timestamp-sanitization.

### File 3 — `.github/workflows/android-app.yml` (new, ~60 lines)

```yaml
name: Android Companion App Build

on:
  push:
    paths:
      - 'android/**'
      - '.github/workflows/android-app.yml'
  pull_request:
    paths:
      - 'android/**'

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '17'
      - name: Cache Gradle
        uses: actions/cache@v4
        with:
          path: |
            ~/.gradle/caches
            ~/.gradle/wrapper
          key: gradle-${{ hashFiles('android/**/*.gradle*', 'android/**/gradle-wrapper.properties') }}
      - name: Generate Gradle wrapper if missing
        run: |
          cd android/SkyHerdCompanion
          if [ ! -f gradlew ]; then
            gradle wrapper --gradle-version 8.5
          fi
      - name: Build unsigned APK
        run: |
          cd android/SkyHerdCompanion
          ./gradlew assembleDebug --no-daemon --stacktrace
      - name: Upload APK
        uses: actions/upload-artifact@v4
        with:
          name: skyherd-companion-android-apk
          path: android/SkyHerdCompanion/app/build/outputs/apk/debug/*.apk
          retention-days: 30
```

### File 4 — `.github/workflows/ios-app.yml` (new, ~55 lines)

```yaml
name: iOS Companion App Build

on:
  push:
    paths:
      - 'ios/**'
      - '.github/workflows/ios-app.yml'
  pull_request:
    paths:
      - 'ios/**'

jobs:
  build:
    runs-on: macos-14
    steps:
      - uses: actions/checkout@v4
      - name: Install XcodeGen
        run: brew install xcodegen
      - name: Generate Xcode project
        run: |
          cd ios/SkyHerdCompanion
          xcodegen generate
      - name: Build (unsigned)
        run: |
          cd ios/SkyHerdCompanion
          xcodebuild \
            -scheme SkyHerdCompanion \
            -configuration Debug \
            -destination 'generic/platform=iOS Simulator' \
            -derivedDataPath build \
            CODE_SIGNING_ALLOWED=NO \
            build
      - name: Upload iOS build products
        uses: actions/upload-artifact@v4
        with:
          name: skyherd-companion-ios-build
          path: ios/SkyHerdCompanion/build/Build/Products/
          retention-days: 30
```

**Fallback plan for H3-05:** if CI fights us for > 60 min (common
brokenness: xcframework missing, Gradle wrapper generation fails in
sandbox, Swift 5.9 not available on macos-14 runner), we commit the
workflow *syntactically valid* and disable it with a `workflow_dispatch`-only
trigger, then log to `deferred-features.md` as MEDIUM. Repo-local build
commands in the runbook still work.

### File 5 — `docs/HARDWARE_H3_RUNBOOK.md` (new, ~260 lines)

Sections:
1. **Prerequisites** — DJI Mavic Air 2 + controller + phone or tablet;
   optional USB-C OTG cable for MAVSDK failover; accounts: DJI
   developer (free), Apple dev (paid, signing); env vars.
2. **Step 1 — Pair companion app** (iOS or Android).
3. **Step 2 — Bind MQTT broker** — companion app connects to
   `mosquitto` URL from env.
4. **Step 3 — First takeoff** — single `python -m skyherd.demo
   --scenario coyote DRONE_BACKEND=mavic` command.
5. **Step 4 — Failover test** — pull USB-C cable mid-mission; verify
   `adapter.failover` event in ledger; mission still completes.
6. **Step 5 — MAVSDK-only fallback** — disable companion app, force
   `DRONE_BACKEND=mavic_direct_mavsdk` (if we expose the flag); same
   mission runs via MAVSDK alone.
7. **Troubleshooting** — known gotchas, signal issues, battery floor.
8. **CI artifacts** — where to download unsigned APK / iOS build from
   GitHub Actions; manual signing notes.
