# Plan 03-02 — ElevenLabs voice-clone QA

## Objective

Ship a lightweight, deterministic QA test that catches regressions in the voice synthesis pipeline. Uses stdlib SHA-256 on the SilentBackend reference output + file-size/duration bands on mocked ElevenLabs audio. Zero new dependencies.

## Context

@.planning/phases/03-voice-hardening-wes-live-twilio-call-elevenlabs-voice-clone-/03-CONTEXT.md
@src/skyherd/voice/tts.py
@tests/voice/test_tts.py

## Tasks

### Task 1 — Reference hash fixture for SilentBackend [type=auto, tdd=true]

<behavior>
- `SilentBackend().synthesize("Reference.")` produces a byte-identical WAV across runs.
- SHA-256 of the output WAV matches a committed hex hash.
- The hash-check test pinpoints regressions in `_write_wav` header logic, silence generation, or sample-rate drift.
</behavior>

<implementation>
- Compute the reference hash: run `SilentBackend().synthesize("Reference.")` once, SHA-256 the resulting WAV bytes.
- Commit the hash as a text file `tests/voice/fixtures/wes_reference_silent.sha256` (single-line hex + newline).
- NOTE: `SilentBackend` is input-text-agnostic (always writes 250ms silence); the hash is the same for any input. The test asserts this invariant explicitly — "regardless of input text, output hash == committed reference".
</implementation>

<tests>
New in `tests/voice/test_voice_clone_qa.py`:
- `test_silent_backend_reference_hash_stable` — synthesizes 3 different inputs, asserts all hash equal AND equal to the committed fixture.
- `test_silent_backend_reference_duration_band` — asserts 240ms ≤ duration ≤ 260ms (computed from WAV header: data_size / byte_rate).
- `test_silent_backend_reference_is_valid_riff` — re-verifies RIFF structure at the QA level (duplicates test_tts but is layered for QA clarity).
</tests>

**Done when:** fixture file committed, 3 tests green.

### Task 2 — ElevenLabs voice-clone QA (mocked audio) [type=auto, tdd=true]

<behavior>
- Test injects a fake `elevenlabs.ElevenLabs` client that returns a known MP3 byte sequence (a pre-computed "hello world"-ish blob).
- `ElevenLabsBackend.synthesize()` converts this to WAV.
- Assert output file:
  - exists and non-empty
  - size in a reasonable band (>= 1KB if fallback raw-bytes path, >= 2KB if pydub path)
  - first 4 bytes are RIFF *or* first 2 bytes are `ID3` / MP3 sync — both acceptable given the pydub-optional converter
- Computes SHA-256 of output; committing the hash as a "voice clone baseline" detects silent drift in `_mp3_to_wav` conversion.
</behavior>

<implementation>
- Build a deterministic fake MP3 byte sequence: use a pre-computed fixture file `tests/voice/fixtures/fake_elevenlabs_mp3.bin` (≥256 bytes of deterministic pseudo-random data; pydub will fail on this — that's fine, the raw-bytes fallback kicks in).
- Fixture MP3 bytes: `bytes(((i * 37 + 11) & 0xFF) for i in range(512))` — no need to ship a real MP3, fallback path handles it.
- In `tests/voice/test_voice_clone_qa.py`:
  ```python
  def _fake_elevenlabs_client(mp3_bytes):
      # builds a minimal elevenlabs module stub
      ...
  ```
- Committed hash: SHA-256 of the WAV that `_mp3_to_wav` produces from these fake MP3 bytes (via raw-bytes fallback since pydub will fail on garbage).
- Assertion chain: call `ElevenLabsBackend(api_key="fake").synthesize("text")`, read bytes, compare SHA-256 to fixture.
</implementation>

<tests>
- `test_elevenlabs_clone_qa_hash` — injects fake elevenlabs, computes hash, compares to fixture.
- `test_elevenlabs_clone_qa_size_band` — asserts `>= 256 bytes` and `<= 10_000 bytes` (sanity).
- `test_elevenlabs_clone_qa_chunks_concatenated_correctly` — synthesize from an iterator yielding multiple chunks, assert concatenation is same as single-chunk case.
</tests>

**Done when:** 3 new tests green, hashes stable across 3 consecutive runs.

### Task 3 — Real-ElevenLabs opt-in smoke [type=auto]

<behavior>
- Env var `ELEVENLABS_CLONE_QA=1` + real `ELEVENLABS_API_KEY` → optional live smoke test runs, asserts:
  - synthesize returns a path
  - file size ≥ 1KB
  - file exists with .wav extension
- Without `ELEVENLABS_CLONE_QA=1` → test is SKIPPED (not failed) so CI stays offline.
- Marked with `@pytest.mark.live` if the marker exists; else `@pytest.mark.skipif(...)` with explicit env check.
</behavior>

<implementation>
- Check if `pytest.ini` or `pyproject.toml` has a `live` marker. If so, use it; if not, use `@pytest.mark.skipif(not os.environ.get("ELEVENLABS_CLONE_QA"), reason="...")`.
- Place in `tests/voice/test_voice_clone_qa.py` as `class TestLiveElevenLabs` for clear opt-in scoping.
- Single test: `test_live_elevenlabs_short_phrase_synthesizes` — "Howdy boss." → wav with size > 1KB.
- Skipped by default; not added to CI.
</implementation>

<tests>

</tests>

**Done when:** test collected (not failing), SKIP marker visible when env unset.
