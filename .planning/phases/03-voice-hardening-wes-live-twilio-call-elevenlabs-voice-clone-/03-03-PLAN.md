### Task 1 — Extend `skyherd-voice demo` CLI to 5 urgency lines [type=auto]

<behavior>
- `skyherd-voice demo` produces one Wes line for each of: log, text, call, emergency, silent.
- Each line is rendered via `render_urgency_call(WesMessage(...))`.
- Output: per-line urgency tag + Wes script + wav path (or "(log-only)" for silent/log).
- Runs in mock mode by default (via Plan 03-01's `SKYHERD_VOICE=mock` conftest... but CLI is not in tests, so respect `SKYHERD_VOICE` if set, default to unset which reads live chain — unless ELEVENLABS_API_KEY is absent, then falls to silent).
- Exit code 0 regardless; no audio playback required.
</behavior>

<implementation>
- Edit `src/skyherd/voice/cli.py::demo` — replace the current 3-line samples with 5 covering each urgency level:
  ```python

    WesMessage(urgency="log", subject="battery swap on gate cam 2"),
    WesMessage(urgency="text", subject="water tank 3 low", context={"tank_id": "Tank 3", "level": "18%"}),
    WesMessage(urgency="call", subject="coyote at the SW fence", context={"location": "SW fence"}),
    WesMessage(urgency="emergency", subject="predator inside herd, south pasture",
               context={"predator": "coyote"}),
  ```

- Add a test `tests/voice/test_cli.py::test_demo_emits_five_urgency_lines` — runs demo via CliRunner, asserts all 5 urgency tags appear in output.
</implementation>

**Done when:** CLI demo runs cleanly, test green.
