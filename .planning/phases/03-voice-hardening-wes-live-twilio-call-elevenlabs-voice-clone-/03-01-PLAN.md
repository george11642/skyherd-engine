### Task 4 ‚Äî Parametrized fallback-chain cascade test [type=auto, tdd=true]

<behavior>
- Mock `ELEVENLABS_API_KEY` present BUT `ElevenLabsBackend.synthesize` raises ‚Üí `render_urgency_call` should NOT crash; should deliver via dashboard-ring with some WAV (from next backend in chain).
- Actually, the current chain is STATIC (resolved at `get_backend()` call time). The cascade test instead verifies that if the **chosen** backend raises at synthesize-time, there's no recovery (accepted behavior). BUT we can test: `_resolve_backend()` correctly picks next-in-chain when the higher-priority indicator is absent ‚Äî the priority-level logic.
- Add a cascade test that walks through all 4 resolution states:
  1. ELEVENLABS_API_KEY set ‚Üí ElevenLabsBackend
  2. ELEVENLABS_API_KEY unset, piper on PATH ‚Üí PiperBackend
  3. ELEVENLABS_API_KEY unset, piper absent, espeak on PATH ‚Üí EspeakBackend
  4. ELEVENLABS_API_KEY unset, piper+espeak absent ‚Üí SilentBackend
- PLUS: when a backend's synthesize() raises, `render_urgency_call` catches the exception and falls back to SilentBackend as a last resort.
</behavior>

<implementation>
- Edit `src/skyherd/voice/call.py::render_urgency_call` ‚Äî wrap `backend.synthesize(script)` in a `try/except Exception` that logs a warning and retries with `SilentBackend()`.
  ```python
  try:
      wav_path = backend.synthesize(script)
  except Exception as exc:  # noqa: BLE001
      logger.warning("TTS backend %s failed: %s: %s ‚Äî falling back to SilentBackend",
                     type(backend).__name__, type(exc).__name__, exc)
      from skyherd.voice.tts import SilentBackend
      wav_path = SilentBackend().synthesize(script)
  ```
- Parametrize `tests/voice/test_get_backend.py::test_resolve_backend_cascade` with `pytest.mark.parametrize` over the 4 states.
- New test `tests/voice/test_call.py::test_synthesize_failure_falls_back_to_silent` ‚Äî inject a backend whose `synthesize` raises, assert result["delivered_to"]=="dashboard-ring" and wav file exists.
</implementation>

<tests>
- `test_resolve_backend_cascade` parametrized 4 ways.
- `test_synthesize_failure_falls_back_to_silent`.
- `test_synthesize_failure_logs_warning`.
</tests>

**Done when:** 6+ new tests green, graceful degrade verified, no unhandled exceptions in any chain state.
