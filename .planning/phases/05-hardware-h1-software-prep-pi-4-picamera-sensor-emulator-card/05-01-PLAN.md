# Plan 05-01 — Edge CLI extension + bootstrap.sh

## Objective

Ship the "one-command Pi bringup" surface: a curl-pipe bootstrap script that ingests a `credentials.json` (wifi + mqtt broker), and extend the existing `skyherd-edge` CLI with subcommands that Plans 05-02 and 05-03 will populate (`picam`, `coyote`). This is the foundation plan — downstream plans plug behavior into the CLI we scaffold here.

## Scope

1. `hardware/pi/bootstrap.sh` — wraps `scripts/provision-edge.sh` with credential ingestion from `/boot/firmware/skyherd-credentials.json` or `/opt/skyherd-credentials.json` (Pi Imager can drop it at boot). Extracts `wifi_ssid`, `wifi_psk`, `mqtt_url`, `ranch_id`, `edge_id`, `trough_ids`; writes `wpa_supplicant.conf` (if not already online) then delegates to `provision-edge.sh`. Idempotent, `set -euo pipefail`.
2. `hardware/pi/credentials.example.json` — annotated template.
3. `hardware/pi/README.md` — 20-line quickstart: "Flash SD card → drop credentials.json on /boot → plug in Pi → done".
4. Extend `src/skyherd/edge/cli.py` with three new subcommands: `picam`, `coyote`, `verify-bootstrap`. Implementations for `picam` / `coyote` are placeholders that call into stubs to be populated in Plans 05-02 / 05-03 respectively. `verify-bootstrap` parses a given `credentials.json` and reports missing fields (used by the bootstrap script to fail-fast before ssh-pipe).
5. Unit tests: `tests/edge/test_cli.py` extended (or new if missing) to cover `verify-bootstrap` happy + sad paths.
6. Bootstrap script tests: `tests/hardware/test_bootstrap_script.py` — runs `bash -n hardware/pi/bootstrap.sh` (syntax check), and runs the script in `--dry-run` mode against fixture credentials.

## Tasks

### Task 1 — [auto] Scaffold `hardware/pi/bootstrap.sh` + example credentials + README

Create the three files in `hardware/pi/`. The shell script must:

- `set -euo pipefail`
- Default to reading `/boot/firmware/skyherd-credentials.json`, accept `SKYHERD_CREDS_FILE` env override.
- Error with a clear message if the file is absent.
- Use `jq` (install via apt if absent) to extract wifi + mqtt fields.
- Call `bash scripts/provision-edge.sh <edge_id> <trough_ids>` with `RANCH_ID=... MQTT_URL=...` exported.
- Optional dry-run mode (`--dry-run`) that prints the derived provisioning command without executing.

**Done when:** `bash -n hardware/pi/bootstrap.sh` succeeds. `bash hardware/pi/bootstrap.sh --dry-run` with a fixture credentials.json prints `DRY-RUN: provision-edge.sh edge-house "trough_1,trough_2"` and exits 0.

### Task 2 — [auto] Extend `skyherd-edge` CLI with `picam`, `coyote`, `verify-bootstrap`

Modify `src/skyherd/edge/cli.py`:

- Add `verify-bootstrap` subcommand: accepts `--credentials-file` (default `/boot/firmware/skyherd-credentials.json`), parses JSON, validates presence of `wifi_ssid`, `wifi_psk`, `mqtt_url`, `ranch_id`, `edge_id`, `trough_ids`, exits 0 if ok, exits 2 with a list of missing fields if not.
- Add `picam` subcommand: placeholder that imports `skyherd.edge.picam_sensor` with `ImportError` handled (module may not exist yet at Plan 05-01 completion). Prints `picam not implemented in Plan 05-01 — run Plan 05-02`. **Plan 05-02 will replace this body.**
- Add `coyote` subcommand: same pattern, placeholder. Plan 05-03 will fill.

**Done when:** `uv run skyherd-edge verify-bootstrap --credentials-file tests/hardware/fixtures/creds_good.json` exits 0. `... creds_bad_missing_mqtt.json` exits 2 with "Missing fields: mqtt_url". `uv run skyherd-edge --help` lists 5 subcommands (run, smoke, picam, coyote, verify-bootstrap).
