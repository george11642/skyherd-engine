<objective>
Convert 9 non-drone, non-voice/call.py silent-except sites from `except ...: pass` to the project-standard `except ... as exc: logger.{warning,debug}(...)` pattern. Addresses **HYG-01** for the sensor / server / voice-tts / edge / scenarios subsystems. Drone sites (f3_inav, sitl_emulator) handled in Plan 04; voice/call.py:200 handled in Plan 01. Category-1 WONTFIX sites (9 `asyncio.CancelledError` / `KeyboardInterrupt` shutdown catches) are intentionally left as-is and documented in SUMMARY.

Purpose: Silent-except is the #1 source of "code is lying to me" bugs — swallowed exceptions hide real failures (MQTT close on broker restart, mp3 decode failure, SSE subscriber-removal races). This phase applies the already-established `rancher_mcp.py:92-100` BLE001+logger pattern to 9 holdout sites. WARNING level for real failures that operators need to see (MQTT close errors); DEBUG level for defensive fallbacks where a safe default is in use (parse errors, optional-dep imports, platform-variant behavior).

Output: 7 source files modified in-place (no structural changes, no new files). 5 caplog tests added to assert the 3 Category-2 WARNING-level sites actually log (coverage-regression guard per 03-RESEARCH.md:697-702).
</objective>
